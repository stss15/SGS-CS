# 7.5 Validation and Verification Assessment Plan

## Goal
Create a self-marking assessment for Topic 7.5 using the standard template.

## Content Source

## Question Design

### Q1: Validation vs Verification (Categorisation)
**Concept**: Distinguishing between the two checks.
-   **Buckets**: Validation, Verification
-   **Items**:
    -   Checking data is reasonable (Validation)
    -   Automated check by computer (Validation)
    -   Range Check (Validation)
    -   Checking data matches source (Verification)
    -   Double Entry (Verification)
    -   Visual Check (Verification)

### Q2: Validation Checks (Matching)
**Concept**: Types of validation.
-   **Pairs**:
    -   Range Check -> Value between upper and lower limits
    -   Length Check -> Exact or reasonable number of characters
    -   Type Check -> Data is of the correct data type (e.g., Integer)
    -   Presence Check -> Data has not been left blank
    -   Format Check -> Data conforms to a pre-defined pattern

### Q3: Check Digit Errors (Multiple Choice)
**Concept**: What errors check digits detect.
-   **Question**: Which error would a check digit likely detect?
-   **Options**:
    -   Entering 5327 instead of 5307 (Incorrect digit) (Correct)
    -   Entering the wrong date of birth
    -   Entering a valid but incorrect product code
    -   Forgetting to save the file

### Q4: Pseudocode Validation (Ordering)
**Concept**: Logic for a range check.
-   **Order**:
    1.  REPEAT
    2.  INPUT Mark
    3.  IF Mark < 0 OR Mark > 100 THEN
    4.  OUTPUT "Error"
    5.  ENDIF
    6.  UNTIL Mark >= 0 AND Mark <= 100

### Q5: Validation Examples (Drag & Drop)
**Concept**: Applying checks to scenarios.
-   **Buckets**: Range Check, Length Check, Type Check
-   **Items**:
    -   Age must be 11-18 (Range)
    -   Month must be 1-12 (Range)
    -   Password must be 8 chars (Length)
    -   Postcode must be 7 chars (Length)
    -   Quantity must be Integer (Type)
    -   Price must be Currency (Type)
