## Question Design

### Q1: Array Definitions (Matching)
**Concept**: Key terms related to arrays.
-   **Pairs**:
    -   Element -> A single data item in an array
    -   Index -> The position of an element
    -   1D Array -> A list of items
    -   2D Array -> A table with rows and columns
    -   Dimension -> The number of indices needed to locate an item

### Q2: Array Indexing (Multiple Choice)
**Concept**: Accessing elements in a 1D array (0-based).
-   **Array**: `Scores = [10, 20, 30, 40, 50]`
-   **Question**: What is the value of `Scores[2]`?
-   **Options**:
    -   30 (Correct)
    -   20
    -   40
    -   10

### Q3: 2D Array Coordinates (Multiple Choice)
**Concept**: Accessing elements in a 2D array.
-   **Grid**:
    Row 0: [A, B, C]
    Row 1: [D, E, F]
    Row 2: [G, H, I]
-   **Question**: What is at `Grid[1, 2]`? (Row 1, Col 2)
-   **Options**:
    -   F (Correct)
    -   E
    -   H
    -   C

### Q4: Populating Arrays (Ordering)
**Concept**: Using a loop to fill an array.
-   **Order**:
    1.  `DECLARE Numbers : ARRAY[0:4] OF INTEGER`
    2.  `FOR i <- 0 TO 4`
    3.  `INPUT Numbers[i]`
    4.  `NEXT i`
    5.  `OUTPUT "Array Full"`

### Q5: Array Characteristics (True/False Grid)
**Concept**: Properties of arrays.
-   **Statements**:
    -   Arrays can store different data types in the same list (False - usually same type in standard definition)
    -   The first index is always 1 (False - usually 0)
    -   Arrays allow random access to elements using an index (True)
    -   2D arrays require nested loops to traverse fully (True)
