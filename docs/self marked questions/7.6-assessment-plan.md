## Question Design

### Q1: Test Data Types (Matching)
**Concept**: Defining the types of test data.
-   **Pairs**:
    -   Normal Data -> Data that should be accepted by the system
    -   Abnormal Data -> Data that should be rejected by the system
    -   Extreme Data -> The largest and smallest values that are accepted
    -   Boundary Data -> Values at the limit of what is accepted and rejected

### Q2: Identifying Test Data (Categorisation)
**Concept**: Classifying data for a range 0-100.
-   **Buckets**: Normal, Abnormal, Extreme, Boundary
-   **Items**:
    -   50 (Normal)
    -   25 (Normal)
    -   -1 (Abnormal)
    -   101 (Abnormal)
    -   "Ten" (Abnormal)
    -   0 (Extreme)
    -   100 (Extreme)
    -   -1 and 0 (Boundary - Lower)
    -   100 and 101 (Boundary - Upper)
    *Note: To avoid confusion in drag & drop, I will simplify categories or items. Let's use specific values for specific types.*
    -   **Revised Buckets**: Normal, Abnormal, Extreme
    -   **Revised Items**:
        -   50 (Normal)
        -   -5 (Abnormal)
        -   150 (Abnormal)
        -   "Five" (Abnormal)
        -   0 (Extreme)
        -   100 (Extreme)

### Q3: Boundary Data Logic (Multiple Choice)
**Concept**: Understanding boundary testing.
-   **Question**: For a range of 1 to 10, which pair of values represents the upper boundary?
-   **Options**:
    -   10 and 11 (Correct)
    -   9 and 10
    -   1 and 0
    -   1 and 2

### Q4: Test Data Application (Drag & Drop)
**Concept**: Applying types to a scenario.
-   **Scenario**: Inputting a month number (1-12).
-   **Buckets**: Accept, Reject
-   **Items**:
    -   6 (Accept - Normal)
    -   1 (Accept - Extreme)
    -   12 (Accept - Extreme)
    -   0 (Reject - Boundary)
    -   13 (Reject - Boundary)
    -   "June" (Reject - Abnormal)

### Q5: Test Data Definitions (Ordering)
**Concept**: Logical order of testing.
-   **Order**:
    1.  Determine the purpose of the algorithm
    2.  Select Normal data to test expected results
    3.  Select Extreme data to test limits
    4.  Select Abnormal data to test error handling
    5.  Compare actual results with expected results
