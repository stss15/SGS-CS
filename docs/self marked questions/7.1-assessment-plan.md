## Question Design

### Q1: PDLC Stages (Ordering)
**Concept**: The correct sequence of the life cycle.
-   **Order**:
    1.  Analysis
    2.  Design
    3.  Coding
    4.  Testing
    5.  Maintenance

### Q2: Analysis Tools (Matching)
**Concept**: Key terms in Analysis.
-   **Pairs**:
    -   Abstraction -> Removing unnecessary details
    -   Decomposition -> Breaking a problem into smaller parts
    -   Requirements Specification -> Defining exactly what is needed

### Q3: Design vs Coding (Categorisation)
**Concept**: Distinguishing between design tools and coding artifacts.
-   **Buckets**: Design, Coding
-   **Items**:
    -   Flowchart (Design)
    -   Pseudocode (Design)
    -   Structure Chart (Design)
    -   Source Code (Coding)
    -   Iterative Testing (Coding)

### Q4: Iterative Testing (Multiple Choice)
**Concept**: Definition of iterative testing.
-   **Question**: What is 'Iterative Testing'?
-   **Options**:
    -   Testing modules repeatedly while writing code (Correct)
    -   Testing the final program once
    -   Testing the hardware only
    -   Designing the program structure

### Q5: PDLC Definitions (Matching)
**Concept**: Purpose of each stage.
-   **Pairs**:
    -   Analysis -> Defining the problem and requirements
    -   Design -> Planning how the program will work
    -   Coding -> Writing the program in a specific language
    -   Testing -> Running the program with test data
