## Question Design

### Q1: Digital vs Fiat Currency (Categorisation)
**Concept**: Differences between traditional and digital money.
-   **Buckets**: Fiat Currency, Digital Currency
-   **Items**:
    -   Physical form (coins/notes) (Fiat)
    -   Backed by governments (Fiat)
    -   Exists purely as data (Digital)
    -   Transferred via apps/online (Digital)
    -   Can be cryptocurrency (Digital)
    -   Legal tender in physical stores (Fiat)

### Q2: Blockchain Components (Matching)
**Concept**: Key terms in blockchain technology.
-   **Pairs**:
    -   Genesis Block -> The first block in a chain
    -   Hash Value -> Unique fingerprint of a block
    -   Miner -> Network user who verifies transactions
    -   Decentralised -> No central server or authority

### Q3: How Blockchain Works (Ordering)
**Concept**: The process of adding a block.
-   **Order**:
    1.  Transaction takes place
    2.  New block is created with data
    3.  Hash value is generated
    4.  Block is sent to network for checking
    5.  Block is added to the chain

### Q4: Cryptocurrency Features (True/False Grid)
**Concept**: Characteristics of crypto.
-   **Statements**:
    -   Cryptocurrency is regulated by a central bank. (False)
    -   Transactions are public and trackable. (True)
    -   It uses a blockchain network for security. (True)
    -   It is impossible to hack a single block without invalidating the chain. (True)

### Q5: Block Structure (Dropdown Gap Fill)
**Concept**: What makes up a block.
-   **Text**:
    -   A block contains (Data) about the transaction.
    -   It has a unique (Hash) value generated by an algorithm.
    -   It also stores the (Previous Hash) to link to the chain.
    -   A (Timestamp) records when the block was created.
