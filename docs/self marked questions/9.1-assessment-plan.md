## Question Design

### Q1: Database Structure (Matching)
**Concept**: Basic database terminology.
-   **Pairs**:
    -   Table -> A collection of related records
    -   Record -> A row containing data about one item
    -   Field -> A column containing one specific piece of data
    -   Primary Key -> A unique identifier for a record
    -   Database -> A structured collection of data

### Q2: Data Types (Drag & Drop)
**Concept**: Identifying data types.
-   **Buckets**: Text, Integer, Real, Boolean, Date
-   **Items**:
    -   "Smith" (Text)
    -   25 (Integer)
    -   15.99 (Real)
    -   TRUE (Boolean)
    -   12/12/2023 (Date)

### Q3: SQL Structure (Ordering)
**Concept**: Correct order of SQL clauses.
-   **Order**:
    1.  `SELECT FirstName, LastName`
    2.  `FROM Students`
    3.  `WHERE Age > 15`
    4.  `ORDER BY LastName`

### Q4: SQL Logic (Multiple Choice)
**Concept**: Interpreting a query.
-   **Table**: `Cars` (Model, Color)
    -   Ford, Red
    -   Toyota, Blue
    -   Honda, Red
-   **Query**: `SELECT Model FROM Cars WHERE Color = 'Red'`
-   **Options**:
    -   Ford, Honda (Correct)
    -   Ford, Toyota, Honda
    -   Toyota
    -   Red, Red

### Q5: SQL Operators (Matching)
**Concept**: Understanding SQL operators.
-   **Pairs**:
    -   `<>` -> Not Equal To
    -   `>=` -> Greater Than or Equal To
    -   `LIKE` -> Pattern Search
    -   `AND` -> Both conditions must be true
    -   `SUM` -> Calculates the total of a field
