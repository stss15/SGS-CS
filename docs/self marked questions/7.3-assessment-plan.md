## Question Design

### Q1: Algorithm Purpose (Multiple Choice)
**Concept**: Identifying what an algorithm does.
-   **Snippet**:
    ```
    INPUT A, B
    IF A > B THEN
       OUTPUT A
    ELSE
       OUTPUT B
    ENDIF
    ```
-   **Question**: What is the purpose of this algorithm?
-   **Options**:
    -   To output the larger of two numbers (Correct)
    -   To add two numbers together
    -   To check if two numbers are equal
    -   To output the smaller of two numbers

### Q2: Trace Output (Multiple Choice)
**Concept**: Predicting the output.
-   **Snippet**: Same as Q1.
-   **Inputs**: A = 7, B = 18.
-   **Question**: What will be output if the inputs are 7 and 18?
-   **Options**:
    -   18 (Correct)
    -   7
    -   25
    -   Error

### Q3: Process Identification (Matching)
**Concept**: Linking code to action.
-   **Pairs**:
    -   `INPUT Name` -> Getting data from the user
    -   `Total <- A + B` -> Performing a calculation
    -   `IF Age > 18` -> Making a decision
    -   `OUTPUT "Hello"` -> Displaying information

### Q4: Algorithm Steps (Ordering)
**Concept**: Logical sequence.
-   **Task**: Calculate area of a rectangle.
-   **Order**:
    1.  INPUT Length
    2.  INPUT Width
    3.  Area <- Length * Width
    4.  OUTPUT Area

### Q5: Variable Tracking (Drag & Drop)
**Concept**: Tracking values.
-   **Snippet**:
    ```
    X <- 5
    Y <- 10
    X <- X + Y
    Y <- X - Y
    ```
-   **Buckets**: Final Value of X, Final Value of Y
-   **Items**:
    -   15 (Final X)
    -   5 (Final Y)
    -   10 (Initial Y)
    -   5 (Initial X)
