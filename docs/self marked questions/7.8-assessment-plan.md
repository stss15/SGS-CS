## Question Design

### Q1: Error Types (Matching)
**Concept**: Distinguishing between syntax and logic errors.
-   **Pairs**:
    -   Syntax Error -> Code does not follow the rules of the language
    -   Logic Error -> Program runs but produces incorrect results
    -   Runtime Error -> Program crashes during execution
    -   Trace Table -> Tool used to find logic errors

### Q2: Logic Error Identification (Multiple Choice)
**Concept**: Spotting the flaw in a loop.
-   **Snippet**:
    ```
    Total <- 0
    FOR X <- 1 TO 5
       Total <- Total * X
    NEXT X
    ```
-   **Question**: Why will Total remain 0?
-   **Options**:
    -   Total is multiplied by X, and 0 * anything is 0 (Correct)
    -   The loop does not run
    -   Total should be initialised to 1
    -   X starts at 1

### Q3: Debugging Strategies (Ordering)
**Concept**: Steps to fix an error.
-   **Order**:
    1.  Run the program with test data
    2.  Compare actual output with expected output
    3.  Identify the discrepancy
    4.  Use a trace table to locate the error in logic
    5.  Correct the code and re-test

### Q4: Correcting Logic (Drag & Drop)
**Concept**: Fixing a min/max algorithm.
-   **Snippet**:
    ```
    Min <- 0
    Max <- 0
    INPUT Num
    IF Num < Min THEN Min <- Num
    IF Num > Max THEN Max <- Num
    ```
-   **Problem**: Min initialized to 0 won't work for positive numbers.
-   **Buckets**: Correct Initialization, Incorrect Initialization
-   **Items**:
    -   Min <- 1000 (Correct)
    -   Min <- First Input (Correct)
    -   Min <- 0 (Incorrect)
    -   Max <- 0 (Correct - assuming positive inputs)
    -   Max <- -1000 (Correct)

### Q5: Boundary Errors (Multiple Choice)
**Concept**: Off-by-one errors.
-   **Snippet**: `FOR X <- 1 TO 10` (Loop runs 10 times)
-   **Question**: If we want the loop to run 10 times, but we write `FOR X <- 0 TO 10`, how many times will it run?
-   **Options**:
    -   11 times (Correct)
    -   10 times
    -   9 times
    -   Infinite times
