# 1.2 Text, Sound and Images - Revision Artifact Prompts

## Concept: Character Sets (ASCII vs Unicode)
- **Type**: Comparison Infographic
- **Description**: A side-by-side comparison card detailing the differences between ASCII and Unicode.
- **Prompt**: Create a split-screen comparison infographic.
  - **Left Side (ASCII)**:
    - Title: "ASCII".
    - Icon: A simple 'A' key or US flag.
    - Bullet points: "7-bit code", "128 Characters", "English Language only", "Small storage size".
  - **Right Side (Unicode)**:
    - Title: "Unicode".
    - Icon: A globe with various symbols (Chinese, Arabic, Emojis).
    - Bullet points: "16-bit / 32-bit code", "65,000+ Characters", "All Global Languages", "Larger storage size".
  - **Style**: Clean, educational, high contrast.

## Concept: Analogue to Digital Sound
- **Type**: Annotated Graph
- **Description**: A graph illustrating the sampling process with technical labels.
- **Prompt**: Create a technical graph showing "Sound Sampling".
  - **Visual**: A smooth red curve labeled "Analogue Wave". Vertical blue bars reaching up to the curve labeled "Digital Samples".
  - **Axes**: X-axis labeled "Time (Sample Rate)". Y-axis labeled "Amplitude (Bit Depth)".
  - **Annotations**:
    - Arrow to the gap between bars: "Sample Rate = Samples per second (Hz)".
    - Arrow to the height levels: "Bit Depth = Bits per sample".
    - Text box: "ADC (Analogue to Digital Converter) measures amplitude at set intervals."

## Concept: Bitmap Images & Colour Depth
- **Type**: Visual Definition Sheet
- **Description**: Explaining pixels and colour depth using a zoomed-in visual.
- **Prompt**: Create a revision sheet for "Bitmap Images".
  - **Visual**: A picture of a simple object (e.g., a smiley face) with a "magnifying glass" zoom revealing the square pixels.
  - **Labels**:
    - Arrow to a square: "Pixel (Picture Element)".
    - Text box: "Binary Data: Each pixel is stored as a binary number."
  - **Colour Depth Section**: Show three single pixels side-by-side.
    - Pixel 1 (B&W): "1-bit (2 colours)".
    - Pixel 2 (256 colours): "8-bit (256 colours)".
    - Pixel 3 (True Colour): "24-bit (16 million colours)".

## Concept: Image Resolution
- **Type**: Comparison Visual
- **Description**: Demonstrating the effect of resolution on image quality.
- **Prompt**: Create a comparison showing "High vs Low Resolution".
  - **Left Image**: A sharp, clear photo of a car wheel. Label: "High Resolution (Many Pixels)". Text: "Large File Size, High Detail".
  - **Right Image**: The same photo but blocky and pixelated. Label: "Low Resolution (Few Pixels)". Text: "Small File Size, Pixelated/Fuzzy".
  - **Definition**: Bottom banner: "Resolution = Width (pixels) x Height (pixels)".
