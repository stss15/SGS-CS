# 3.3 Data Storage - Revision Artifact Prompts

## Concept: Memory Hierarchy
- **Type**: Pyramid Diagram
- **Description**: Showing the speed and size relationship of memory types.
- **Prompt**: Create a "Memory Hierarchy Pyramid".
  - **Top (Fastest/Smallest)**: "Registers (CPU)".
  - **Second Layer**: "Cache (L1/L2/L3)". Text: "SRAM. Very Fast."
  - **Third Layer**: "Primary Memory (RAM)". Text: "DRAM. Fast. Volatile."
  - **Bottom (Slowest/Largest)**: "Secondary Storage (HDD/SSD)". Text: "Non-Volatile. Permanent."
  - **Arrows**: Arrow up side labeled "Increasing Speed". Arrow down side labeled "Increasing Capacity".

## Concept: RAM vs ROM
- **Type**: Comparison Table
- **Description**: Key differences between Random Access Memory and Read Only Memory.
- **Prompt**: Create a comparison card for "RAM vs ROM".
  - **RAM**:
    - "Volatile (Data lost when power off)."
    - "Read and Write."
    - "Stores: OS, Running Programs, Data in use."
    - "Larger capacity (e.g., 8GB+)."
  - **ROM**:
    - "Non-Volatile (Data kept when power off)."
    - "Read Only (Cannot be changed easily)."
    - "Stores: BIOS / Bootstrap Loader."
    - "Small capacity (e.g., 4MB)."

## Concept: Magnetic vs Solid State Storage
- **Type**: Technical Comparison
- **Description**: How HDD and SSD work.
- **Prompt**: Create a split visual comparing "HDD vs SSD".
  - **HDD (Magnetic)**:
    - Visual: Spinning platters with a read/write arm.
    - Text: "Moving parts. Magnetic platters. Read/Write head. High latency. Fragile. Cheaper per GB."
  - **SSD (Solid State)**:
    - Visual: Chips on a board (NAND Flash).
    - Text: "No moving parts. Flash memory (Floating Gate Transistors). Low latency (Fast). Durable. Low power. More expensive."

## Concept: Optical Media (CD, DVD, Blu-ray)
- **Type**: Comparison Chart
- **Description**: Differences in capacity and technology.
- **Prompt**: Create a comparison chart for "Optical Storage".
  - **CD**: "Red Laser (780nm). Capacity: ~700MB. Use: Audio."
  - **DVD**: "Red Laser (650nm). Capacity: ~4.7GB (Single Layer). Use: Standard Def Video."
  - **Blu-ray**: "Blue/Violet Laser (405nm). Capacity: ~25GB (Single Layer). Use: High Def Video."
  - **Note**: "Shorter wavelength = Smaller pits = More data."

## Concept: Virtual Memory
- **Type**: Process Diagram
- **Description**: How the OS handles running out of RAM.
- **Prompt**: Create a diagram explaining "Virtual Memory".
  - **Scene**: "RAM is Full".
  - **Action**: Arrow from RAM to HDD labeled "Page Out (Swap)". Text: "Move least used data to HDD (Swap File)."
  - **Action**: Arrow from HDD to RAM labeled "Page In". Text: "Load needed data back to RAM."
  - **Result**: "Illusion of unlimited memory. Performance slows down (Disk Thrashing)."

## Concept: Cloud Storage
- **Type**: Concept Map
- **Description**: Storing data on remote servers.
- **Prompt**: Create a concept map for "Cloud Storage".
  - **Central Icon**: Cloud.
  - **Definition**: "Data stored on remote servers accessed via Internet."
  - **Pros**: "Access anywhere. Auto-backup. Easy sharing. Scalable."
  - **Cons**: "Needs Internet. Security risks (Hacking). Monthly cost. Control of data."
  - **Types**: "Public (Dropbox), Private (Company), Hybrid."
