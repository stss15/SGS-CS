# 7.6 Test Data - Revision Artifact Prompts

## Concept: Types of Test Data
- **Type**: Visual Spectrum / Target Diagram
- **Description**: Categorizing data used for testing.
- **Prompt**: Create a visual guide for "Types of Test Data".
  - **Scenario**: "Accepting marks between 0 and 100."
  - **Normal Data (Valid)**:
    - Color: Green.
    - Text: "Typical, expected values."
    - Example: "50, 23, 89."
  - **Boundary Data (Edge Cases)**:
    - Color: Orange/Yellow.
    - Text: "Values at the very edge of the range."
    - Example: "0, 100." (Sometimes -1 and 101 are included as boundary/invalid).
  - **Abnormal / Erroneous Data (Invalid)**:
    - Color: Red.
    - Text: "Values that should be rejected."
    - Example: "-5, 150, 'Ten' (Wrong Type)."
  - **Extreme Data**: (Optional, similar to Boundary). "Largest and smallest VALID values (0, 100)."
