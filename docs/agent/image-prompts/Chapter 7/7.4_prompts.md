# 7.4 Standard Methods of Solution - Revision Artifact Prompts

## Concept: Totalling vs Counting
- **Type**: Comparison Visual
- **Description**: Distinguishing between adding values and counting occurrences.
- **Prompt**: Create a comparison of "Totalling vs. Counting".
  - **Totalling**:
    - Visual: A piggy bank. Coins ($1, $2, $5) are being dropped in.
    - Code: `Total ← Total + Number`
    - Meaning: "Adding the VALUE of the item to the running total."
  - **Counting**:
    - Visual: A clicker counter (like a doorman uses).
    - Code: `Count ← Count + 1`
    - Meaning: "Adding ONE for each item/event."

## Concept: Finding Maximum and Minimum
- **Type**: Process Visual / Analogy
- **Description**: How an algorithm finds the highest/lowest value.
- **Prompt**: Create a visual for "Finding the Maximum".
  - **Analogy**: "King of the Hill".
  - **Step 1**: "Assume the first person is King." (`Max ← List[1]`)
  - **Step 2**: "Challenger appears." (Compare `List[2]` with `Max`).
  - **Step 3**: "If Challenger is stronger (>), they become King." (`IF List[i] > Max THEN Max ← List[i]`)
  - **Step 4**: "Repeat for all challengers."

## Concept: Linear Search
- **Type**: Step-by-Step Diagram
- **Description**: Visualizing the sequential search process.
- **Prompt**: Create a diagram for "Linear Search".
  - **Scenario**: Looking for the number "42" in a list: `[10, 5, 42, 8]`.
  - **Step 1**: Check Index 1 (10). "Is 10 = 42? No." (Move Next).
  - **Step 2**: Check Index 2 (5). "Is 5 = 42? No." (Move Next).
  - **Step 3**: Check Index 3 (42). "Is 42 = 42? YES!" (Found).
  - **Output**: "Found at Position 3."
  - **Pros/Cons**: "Simple but slow for long lists."

## Concept: Bubble Sort
- **Type**: Process Diagram
- **Description**: Visualizing the sorting mechanism.
- **Prompt**: Create a diagram for "Bubble Sort Pass 1".
  - **List**: `[5, 2, 8, 1]`.
  - **Compare 1 & 2**: `5` vs `2`. "5 > 2? Yes. SWAP." -> `[2, 5, 8, 1]`.
  - **Compare 2 & 3**: `5` vs `8`. "5 > 8? No. No Swap." -> `[2, 5, 8, 1]`.
  - **Compare 3 & 4**: `8` vs `1`. "8 > 1? Yes. SWAP." -> `[2, 5, 1, 8]`.
  - **Result**: "Largest value (8) has 'bubbled' to the end."
  - **Note**: "Repeat passes until no swaps are needed."
