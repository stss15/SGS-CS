# 7.7 Trace Tables - Revision Artifact Prompts

## Concept: How to Trace an Algorithm
- **Type**: Annotated Table
- **Description**: A guide to completing a trace table (Dry Run).
- **Prompt**: Create an annotated "Trace Table" example.
  - **The Algorithm**: A simple loop:
    ```
    X ← 0
    FOR I ← 1 TO 3
      X ← X + 2
      OUTPUT X
    NEXT I
    ```
  - **The Table**:
    - **Header Row**: Columns for `I`, `X`, and `OUTPUT`.
    - **Row 1**: `I` is empty. `X` becomes `0`.
    - **Row 2**: `I` becomes `1`. `X` becomes `2` (0+2). Output `2`.
    - **Row 3**: `I` becomes `2`. `X` becomes `4` (2+2). Output `4`.
    - **Row 4**: `I` becomes `3`. `X` becomes `6` (4+2). Output `6`.
  - **Annotations**:
    - Arrow pointing to a cell: "Update value ONLY when it changes."
    - Arrow pointing to Output: "Write exactly what is printed."
    - Note: "Used to find Logic Errors."
