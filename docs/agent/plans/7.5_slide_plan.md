# Slide Plan: 7.5 Validation and Verification

**Goal:** Separate “Is it sensible?” (Validation) from “Is it the same as the source?” (Verification) and apply every required check type with clear examples.

**Pedagogy:** Two guardians: the Bouncer (Validation) and the Mirror (Verification). Each check is taught with message wording, allowed/rejected examples, and a re-entry loop.

## 2. Hook / Big Question
- **Type:** Concept (Horizontal)
- **Content:**
  - **Question:** How do we stop garbage from entering the system?
  - **Visual:** “GIGO” chute with a bouncer stopping trash.
  - **Key Idea:** Validation blocks unreasonable data; Verification ensures faithful copying.

## 3. Validation vs Verification
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** Two Different Jobs
  - **Validation:** Is the data reasonable? (Range, Length, Type, Presence, Format, Check Digit.)
  - **Verification:** Does the entered data match the original? (Double-entry or visual check.)
  - **Common pitfall:** Validation cannot spot a correctly-typed but wrong value; verification cannot judge reasonableness.

## 4. Validation Toolkit
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** Six Checks You Must Know
  - Range, Length, Type, Presence, Format, Check Digit.

### 4.1 Vertical Deep Dive: Range, Length, Type
- **Type:** Vertical Stack
- **Slide 1 (Range Check):**
  - Accept only within limits. Example: 0–100% marks. Re-prompt until valid.
- **Slide 2 (Length Check):**
  - Exact length (8-char password) or range (2–30 letters for surname).
- **Slide 3 (Type Check):**
  - Must be integer for “NumberOfBrothers”. Reject “three”.
  - **Note:** Use integer test (e.g., DIV trick) in pseudocode.

### 4.2 Vertical Deep Dive: Presence, Format, Check Digit
- **Type:** Vertical Stack
- **Slide 1 (Presence Check):**
  - Field cannot be blank (email field marked *).
- **Slide 2 (Format Check):**
  - Pattern required (DD/MM/YYYY, postcode shape, CUB9999).
- **Slide 3 (Check Digit):**
  - Final digit calculated from previous digits (ISBN, barcodes) to spot typos/transpositions.

## 5. Verification Methods
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** Making Sure It Matches
  - **Double Entry:** Type twice (password change).
  - **Visual Check:** Human compares on-screen vs paper/original.
  - **Note:** Verification assumes validation already passed.

## 6. Guided Coding Snippets
- **Type:** Vertical Stack
- **Slide 1 (Range Check Pseudocode):**
    ```
    REPEAT
      INPUT Mark
      IF Mark < 0 OR Mark > 100 THEN
        OUTPUT "Mark must be 0-100. Re-enter."
      ENDIF
    UNTIL Mark >= 0 AND Mark <= 100
    ```
- **Slide 2 (Length Check Pseudocode):**
    ```
    REPEAT
      INPUT Password
      IF LENGTH(Password) <> 8 THEN
        OUTPUT "8 characters needed."
      ENDIF
    UNTIL LENGTH(Password) = 8
    ```
- **Slide 3 (Verification - Double Entry):**
    ```
    INPUT NewPassword
    INPUT ConfirmPassword
    IF NewPassword = ConfirmPassword THEN
      OUTPUT "Saved"
    ELSE
      OUTPUT "Mismatch - try again"
    ENDIF
    ```

## 7. Plenary: Name the Check
- **Type:** Quiz (Vertical Stack)
- **Slide 1:** User leaves postcode blank. Which check stops this? **Answer:** Presence.
- **Slide 2:** Validating a date field shaped DD/MM/YYYY. **Answer:** Format (and Range on year).
- **Slide 3:** Typing a new password twice. **Answer:** Verification (Double entry).
- **Slide 4:** Ensuring mark is whole number 0–100. **Answer:** Type + Range.
