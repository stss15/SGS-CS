# 10.2 Logic Circuits Slide Plan

## 1. Title Slide
- **Slide Type**: Title
- **Content**:
    - Main Title: 10.2 Logic Circuits
    - Sub-title: Topic 10: Boolean Logic | Combining the Pieces
- **Visuals**: Standard title slide layout.

## 2. Hook
- **Slide Type**: Concept
- **Question**: Can one gate do it all?
- **Content**:
    - **The Mystery**: A single AND gate can't control a washing machine.
    - **The Reality**: We combine gates to form **Logic Circuits**.
    - **Key Idea**: Complex problems require chains of logic.
- **Visuals**:
    - Box Blue: The Mystery.
    - Box: The Reality.
    - Fragment: Key Idea.

## 3. From Problem to Circuit
- **Slide Type**: Concept
- **Question**: How do we design a circuit?
- **Content**:
    - We start with a **Problem Statement**.
    - *Example*: "The alarm (X) rings if the Door is open (D) AND the Code is NOT entered (C)."
    - **Logic Expression**: X = D AND (NOT C).
- **Visuals**:
    - Deep Dive arrow: "Building the Circuit".

### Vertical: Drawing the Circuit
- **Slide Type**: Vertical Deep Dive
- **Content**:
    - **Step 1**: Identify Inputs (D, C).
    - **Step 2**: Identify Logic (NOT C, then AND).
    - **Step 3**: Draw the gates.
    - **Visual**:
        - Input C -> NOT gate.
        - Output of NOT + Input D -> AND gate.
        - Output -> X.

## 4. From Expression to Circuit
- **Slide Type**: Concept
- **Question**: How do we read logic algebra?
- **Content**:
    - **Order of Operations**: Brackets first!
    - *Expression*: `(A OR B) AND C`
    - This means: Do `A OR B` first, then take that result and `AND` it with `C`.
- **Visuals**:
    - Deep Dive arrow: "Step-by-Step Build".

### Vertical: Step-by-Step
- **Slide Type**: Vertical Deep Dive
- **Content**:
    - **Gate 1**: A, B go into OR gate.
    - **Gate 2**: Output of OR gate + C go into AND gate.
    - **Final Output**: X.

## 5. Complex Circuits
- **Slide Type**: Concept
- **Question**: What about 3 inputs?
- **Content**:
    - Circuits can have many inputs (A, B, C, D...).
    - We use **Intermediate Points** (P, Q, R) to label the wires between gates.
    - This helps us trace the logic.
- **Visuals**:
    - Diagram of a 3-input circuit with labeled intermediate wires.

## 6. Plenary
- **Slide Type**: Plenary (True or False)
- **Content**:
    1. "We should draw logic gates in any order." (False - follow the brackets/logic).
    2. "Intermediate points help us trace the circuit." (True).
    3. "A logic circuit can only have 2 inputs." (False - can have many).
    4. "NOT C AND D is the same as NOT (C AND D)." (False - brackets change the meaning).
