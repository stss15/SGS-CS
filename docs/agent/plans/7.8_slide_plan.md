# Slide Plan: 7.8 Identifying Errors

**Goal:** Use trace tables and expected results to locate logic errors, then choose the right fix (initialisation, condition, loop bounds, operator).

**Pedagogy:** Bug Hunt. Run a trace, spot where actual output diverges, and repair using standard fixes. Builds directly on 7.7 trace-table skills.

## 2. Hook / Big Question
- **Type:** Concept (Horizontal)
- **Content:**
  - **Question:** The program ran, but the answer is wrong. Now what?
  - **Visual:** Robot walks into a wall because a sensor condition is reversed.
  - **Key Idea:** Logic errors don’t crash code; they give wrong results. We need to find and fix them.

## 3. Error-Hunting Protocol
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** Compare & Pinpoint
  - Steps:
    1. Know the expected result.
    2. Trace with test data; record actual results.
    3. Find the first step where actual ≠ expected.
    4. Decide which mistake type fits (init, condition, update, operator).

### 3.1 Vertical Deep Dive: Common Logic Errors
- **Type:** Vertical Stack
- **Slide 1 (Bad Initialisation):**
  - Example: `Min <- 0` when all data are positive → min never updates.
  - **Fix:** Use first input (`Min <- List[1]`) or a very large starter.
- **Slide 2 (Wrong Condition):**
  - Example: `IF Age > 18 THEN OUTPUT "Child"` → comparison reversed.
  - **Fix:** Swap to `<` or adjust boundary.
- **Slide 3 (Loop Bound / Update Error):**
  - Example: `FOR i <- 1 TO 5 ... NEXT x` or `UNTIL Count = 10` when Count starts at 1 → off-by-one.
  - **Fix:** Align `NEXT` variable and boundary with start value.

## 4. Worked Debug Example
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** Broken Minimum Finder
  - Scenario: Find smallest of 10 numbers; code sets `Min <- 0`.
  - Trace with data: 5, 10, 2 → output 0 (wrong).
  - Fix shown: `Min <- FirstInput`, loop from second value (index 2 because lists start at 1).

## 5. Fix Patterns (Skill Loop)
- **Type:** Vertical Stack
- **Slide 1 (Initialise from Input):**
  - Set `Max` and `Min` to `List[1]`; start loop at 2.
- **Slide 2 (Correct Operator):**
  - Summing uses `+`, counting uses `+ 1`, searching uses comparison `=`.
- **Slide 3 (Correct Loop Exit):**
  - Sentinel loop: `REPEAT ... UNTIL Mark = -1` (will run at least once).
  - For-counted loop: match end value to list length (1-based).

## 6. Plenary: Spot the Error
- **Type:** Quiz (Vertical Stack)
- **Slide 1:** `Total <- Total * Number` for summing. **Fix:** Use `+`.
- **Slide 2:** `FOR i <- 1 TO 5 ... NEXT x`. **Fix:** `NEXT i`.
- **Slide 3:** `IF Score >= 50 THEN OUTPUT "Fail" ELSE OUTPUT "Pass"`. **Fix:** Operator reversed; should be `< 50` for fail or rename messages.
