# 9.2 Structured Query Language (SQL) Slide Plan

## 1. Title Slide
- **Slide Type**: Title
- **Title**: 9.2 Structured Query Language (SQL)
- **Subtitle**: Topic 9: Databases | Talking to the Machine

## 2. Hook
- **Slide Type**: Hook
- **Question**: How do you find a needle in a haystack?
- **Content**:
    - If the haystack is a database, you just ask for it.
    - **SQL** is the language we use to ask questions.
    - It's the most popular database language in the world.
- **Visuals**: Icon of a magnifying glass over a database.

## 3. The Basics
- **Slide Type**: Concept (Vertical Stack)
- **Question**: What are the magic words?
- **Content**:
    - **SELECT**: Which columns do you want?
    - **FROM**: Which table are they in?
    - **WHERE**: Which rows do you want?

### Vertical: The Syntax
- **Slide Type**: Code
- **Content**:
    ```sql
    SELECT FirstName, FamilyName
    FROM PATIENT
    WHERE WardNumber = 7;
    ```
    - **Note**: SQL is not case sensitive, but we write keywords in CAPS.
    - **Quotes**: Text needs quotes (`'Red'`). Numbers don't (`7`).

## 4. Complex Queries
- **Slide Type**: Concept (Vertical Stack)
- **Question**: How do we be more specific?
- **Content**:
    - **AND**: Both conditions must be true.
    - **OR**: Either condition can be true.
    - **NOT**: The condition must be false.

### Vertical: Examples
- **Slide Type**: Example
- **Content**:
    - **AND**: `WHERE Age > 18 AND House = 'Red'`
    - **OR**: `WHERE City = 'London' OR City = 'Paris'`
    - **Wildcard**: `SELECT *` (Select All Columns).

## 5. Sorting and Counting
- **Slide Type**: Concept (Vertical Stack)
- **Question**: How do we organize the mess?
- **Content**:
    - **ORDER BY**: Sorts the results.
    - **ASC**: Ascending (Default).
    - **DESC**: Descending.
    - **COUNT**: Counts how many records match.
    - **SUM**: Adds up a column of numbers.

### Vertical: Examples
- **Slide Type**: Example
- **Content**:
    - `ORDER BY FamilyName ASC` (A-Z)
    - `ORDER BY Score DESC` (High to Low)
    - `SELECT COUNT(ID) FROM Students WHERE House='Blue'`

## 6. Plenary
- **Slide Type**: Plenary (True/False)
- **Content**:
    1. "SELECT * selects all rows." (False - it selects all *columns*).
    2. "ORDER BY default is Ascending." (True).
    3. "You need quotes around numbers in SQL." (False).
    4. "AND means both conditions must be true." (True).
