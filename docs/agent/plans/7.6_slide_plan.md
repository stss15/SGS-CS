# Slide Plan: 7.6 Test Data

**Goal:** Select and justify Normal, Abnormal (Erroneous), Extreme, and Boundary test data to prove algorithms accept what they should and reject what they should.

**Pedagogy:** Crash-test approach. For each data type: definition → example set → expected result. Learners build a mini test table for a live scenario.

---

## 1. Title Slide
- **Type:** Title
- **Content:**
  - Main Title: 7.6 Test Data
  - Sub-title: Topic 7: Algorithm Design and Problem Solving
- **Notes:** Standard title layout.

## 2. Hook / Big Question
- **Type:** Concept (Horizontal)
- **Content:**
  - **Question:** How do you know it won't break?
  - **Visual:** Bridge stress test truck vs collapsed bridge.
  - **Key Idea:** One run isn’t enough. We need specific categories of data.

## 3. The Four Categories
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** Normal, Abnormal, Extreme, Boundary
  - **Definitions:**
    - **Normal:** Typical, should be accepted.
    - **Abnormal/Erroneous:** Wrong type or out of range, must be rejected.
    - **Extreme:** The very limits of the valid range.
    - **Boundary:** The limits **and** just outside them (tests `<` vs `<=`).

### 3.1 Vertical Deep Dive: Normal & Abnormal
- **Type:** Vertical Stack
- **Slide 1 (Normal):**
  - Range 0–100 example: 50, 23, 89 → expect **accepted**.
- **Slide 2 (Abnormal):**
  - Range 0–100 example: -5, 150, "Ten" → expect **rejected with message**.

### 3.2 Vertical Deep Dive: Extreme & Boundary
- **Type:** Vertical Stack
- **Slide 1 (Extreme):**
  - Exact edges of valid range. Example: 0 and 100 → expect **accepted**.
- **Slide 2 (Boundary):**
  - Edge and edge ±1. Example: -1 (reject), 0 (accept), 100 (accept), 101 (reject).
  - **Why:** Reveals off-by-one errors in conditions.

## 4. Build a Test Table (Guided)
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** Scenario: Ticket Sales (1–25 tickets)
  - Learners fill a table with data + type + expected result:
    - 0 (Boundary reject), 1 (Normal accept), 9 (Normal accept), 10 (Boundary accept), 25 (Extreme accept), 26 (Boundary reject), "five" (Abnormal reject).

## 5. Independent Practice (You Do)
- **Type:** Vertical Stack
- **Slide 1:** New scenario: Password length 8 exactly.
  - Provide four test values; classify and give expected result.
- **Slide 2:** New scenario: Age 18–65 inclusive.
  - Provide boundary set (-1, 0, 18, 65, 66) and ask learners to mark accept/reject.

## 6. Plenary: Quickfire Classify
- **Type:** Quiz (Vertical Stack)
- **Slide 1:** Data = 30 for range 18–65. **Answer:** Normal.
- **Slide 2:** Data = 17 for range 18–65. **Answer:** Boundary reject.
- **Slide 3:** Data = 65. **Answer:** Extreme (and Boundary accept).
- **Slide 4:** Data = "Twenty". **Answer:** Abnormal (wrong type).
