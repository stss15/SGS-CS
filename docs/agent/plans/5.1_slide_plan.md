# Slide Deck Plan: 5.1 The Internet and the World Wide Web

## 1. Title Slide
- **Title**: 5.1 The Internet and the World Wide Web (WWW)
- **Subtitle**: Understanding the Infrastructure of the Web

## 2. Internet vs. WWW (The Big Distinction)
- **Slide Type**: Question / Concept
- **Question**: Are the "Internet" and the "World Wide Web" the same thing?
- **Content**:
    - **Internet**: The physical infrastructure. A worldwide collection of interconnected networks (INTERconnected NETwork). Hardware, cables, routers.
    - **WWW**: A service that runs ON the internet. A collection of multimedia web pages and information accessed via web browsers. Uses HTTP/HTTPS.
- **Key Visual/Table**: Comparison table (Infrastructure vs. Information).

## 3. Uniform Resource Locators (URLs)
- **Slide Type**: Breakdown
- **Question**: How do we find a specific page on the web?
- **Content**: Breakdown of `https://www.hoddereducation.co.uk/ict`
    - **Protocol**: `https` (Hypertext Transfer Protocol Secure)
    - **Domain Host**: `www` (World Wide Web)
    - **Domain Name**: `hoddereducation` (The website name)
    - **Domain Type**: `.co.uk` (Organization type/Country code)
    - **Path/File**: `/ict` (Specific page or folder)

## 4. HTTP and HTTPS
- **Slide Type**: Comparison
- **Question**: Why is there a padlock on some websites?
- **Content**:
    - **HTTP**: Hypertext Transfer Protocol. Rules for transferring files.
    - **HTTPS**: Secure version (SSL/TLS). Encrypts data.
    - **Indicator**: Green padlock in the browser bar.

## 5. Web Browsers
- **Slide Type**: Function & Features
- **Question**: What does a web browser actually do?
- **Content**:
    - **Function**: Software to access and display web pages. Interprets (translates) HTML from web servers into visible content (text, images, video).
    - **Features**:
        - Home page
        - Bookmarks/Favourites
        - History
        - Tabs (multitasking)
        - Address bar
        - Cookies management

## 6. Retrieving Web Pages (The DNS Journey)
- **Slide Type**: Vertical Step-by-Step (Deep Dive)
- **Question**: What happens when you type a URL and press Enter?
- **Steps**:
    1.  **Request**: Browser sends URL to Domain Name Server (DNS).
    2.  **Lookup**: DNS looks up the IP address for that domain. (If not found, asks another DNS).
    3.  **Return**: DNS sends the IP address back to the computer.
    4.  **Connect**: Browser uses the IP address to contact the web server.
    5.  **Render**: Web server sends HTML; browser displays the page.

## 7. Cookies
- **Slide Type**: Definition & Types
- **Question**: Why do websites "remember" you?
- **Content**:
    - **Definition**: Small text files stored on YOUR computer by a website.
    - **Purpose**: Track user activity, maintain preferences (language, currency), keep you logged in.
    - **Types**:
        - **Session Cookies**: Temporary. Stored in memory. Deleted when browser closes. (e.g., Shopping basket).
        - **Persistent Cookies**: Permanent. Stored on hard drive. Remain until expiry or deletion. (e.g., "Remember me" login).

## 8. Uses of Cookies
- **Slide Type**: List/Summary
- **Content**:
    - Storing passwords/login details (Persistent).
    - Virtual shopping baskets (Session).
    - Tracking browsing habits for targeted advertising.
    - Customizing web pages (User preferences).
    - Analytics (User history).

## 9. Plenary: True or False?
- **Slide Type**: Vertical Misconception Check
- **Questions**:
    1.  "The Internet and the World Wide Web are the same thing." -> **False**. (Internet is hardware/infrastructure; WWW is the pages/info).
    2.  "Session cookies are stored on your hard drive forever." -> **False**. (Stored in RAM, deleted when browser closes).
    3.  "DNS converts an IP address into a URL." -> **False**. (It converts a URL *into* an IP address).
