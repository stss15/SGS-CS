# Slide Plan: 7.7 Trace Tables

**Goal:** Dry-run algorithms systematically using trace tables to record variable changes and outputs. Build confidence to do this in exams with clear, neat working.

**Pedagogy:** Slow-motion replay. Model one complete trace, then scaffold learners to trace new snippets. Emphasise 1-based loops and exact outputs.

- **Type:** Title
- **Content:**
  - Main Title: 7.7 Trace Tables

- **Notes:** Standard title layout.

## 2. Hook / Big Question
- **Type:** Concept (Horizontal)
- **Content:**
  - **Question:** How do you debug code in your head?
  - **Visual:** Freeze-frame of gears turning; zoom on one gear.
  - **Key Idea:** You become the computer, step through every line, and record changes.

## 3. Trace Table Method
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** 4 Steps
  - Make columns for every variable + Output.
  - Start row with initial values.
  - Add a new row each time a variable changes or an output happens.
  - Use the given test data in order; arrays start at 1.

### 3.1 Vertical Deep Dive: Max/Min Flowchart Example
- **Type:** Vertical Stack
- **Slide 1 (Setup):**
  - Variables: `A`, `B`, `C`, `X`, `Output`.
  - Start: `A=0`, `B=0`, `C=100`.
- **Slide 2 (Iteration 1 with X=9):**
  - `X > B`? Yes → `B <- 9`.
  - `X < C`? Yes → `C <- 9`.
  - `A <- A + 1` → `A=1`.
  - Record row.
- **Slide 3 (Continue):**
  - Step through the provided list (10 values). Show how output row is added at end (`B`, `C`).
- **Slide 4 (Purpose Check):**
  - From the completed table, state purpose: finds largest (B) and smallest (C) of 10 positive numbers.

### 3.2 Vertical Deep Dive: Pseudocode Trace (Array Loop)
- **Type:** Vertical Stack
- **Slide 1 (Code):**
  - `Total <- 0`
  - `FOR i <- 1 TO 3`
  - `INPUT Mark[i]`
  - `Total <- Total + Mark[i]`
  - `NEXT i`
  - `OUTPUT Total`
- **Slide 2 (Trace with Data 10, 20, 30):**
  - Show rows for `i`, `Mark[i]`, `Total`, `Output`.
  - Final `Total=60` output noted.

## 4. Trace Table Tips (Exam Mode)
- **Type:** Concept (Horizontal)
- **Content:**
  - One row per iteration.
  - Copy prompts exactly; outputs should match code (no extra quotes).
  - Watch conditions: `>` vs `>=`, `<` vs `<=`.
  - Keep layout neat; crossed-out values are acceptable if readable.

## 5. Practice Rounds
- **Type:** Vertical Stack
- **Slide 1 (Short Trace):**
  - Code: `X <- 5; Y <- X + 2; OUTPUT Y` → record.
- **Slide 2 (Loop Trace):**
  - Code: `FOR i <- 1 TO 3: Total <- Total + i` (Total starts at 0). Expected final Total = 6.
- **Slide 3 (Boundary Check Trace):**
  - Code uses `REPEAT ... UNTIL Mark = -1`; learners trace with inputs 10, 20, -1.

## 6. Plenary: Quick Identify
- **Type:** Quiz (Vertical Stack)
- **Slide 1:** Which column records printed values? **Answer:** Output column.
- **Slide 2:** If an array starts at 1, what is the last index of 10 items? **Answer:** 10.
- **Slide 3:** What triggers a new row? **Answer:** A variable change or an output.
