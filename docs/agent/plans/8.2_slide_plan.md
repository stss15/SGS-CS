# 8.2 Arrays - Slide Plan

## 1. Title Slide
- **Title**: 8.2 Arrays
- **Subtitle**: Topic 8: Programming | Storing Lists of Data

## 2. Retrieval Practice
- **Slide Type**: Retrieval Practice
- **Question**: Recall Programming Concepts (8.1)
- **Content**:
    1.  What loop would you use to repeat a set number of times? (FOR)
    2.  What data type is "Hello"? (String)
    3.  What is the output of `SUBSTRING("Computer", 1, 3)`? ("Com")

## 3. The Hook
- **Slide Type**: The Hook
- **Question**: How do we store a list of 1000 names?
- **Scenario**: Imagine declaring `Name1`, `Name2`, `Name3`... up to `Name1000`. It would be a nightmare to write and manage.
- **Concept**: An **Array** allows us to store multiple items under one name, accessed by an index.
- **Key Idea**: Arrays are like a row of lockers, each with a number.

## 4. One-Dimensional Arrays
- **Slide Type**: Concept
- **Question**: What is a 1D Array?
- **Analogy**: A pill organiser box with days of the week (Monday, Tuesday...).
- **Definition**: A data structure containing several elements of the same data type, accessed by an index.
- **Visual**:
    ```
    Index:  [1]   [2]   [3]   [4]
    Value:  "Ali" "Bob" "Cat" "Dan"
    ```

## 5. Declaring & Using Arrays
- **Slide Type**: Vertical Deep Dive
- **Question**: How do we use them in code?
- **Content**:
    -   **Declaration**: `DECLARE StudentNames : ARRAY[1:30] OF STRING`
    -   **Assignment**: `StudentNames[1] ← "Ali"`
    -   **Accessing**: `OUTPUT StudentNames[1]`

## 6. Iterating Through Arrays
- **Slide Type**: Concept
- **Question**: How do we process the whole list?
- **Analogy**: The postman delivering mail to house 1, then house 2, then house 3...
- **Content**:
    -   Use a **FOR Loop**.
    -   Example:
        ```
        FOR i ← 1 TO 30
            OUTPUT StudentNames[i]
        NEXT i
        ```

## 7. Two-Dimensional Arrays
- **Slide Type**: Concept
- **Question**: What if we have rows and columns?
- **Analogy**: A spreadsheet or a cinema seating plan (Row 1, Seat 5).
- **Definition**: An array with two indices (Row, Column).
- **Visual**: Grid showing `[Row, Column]`.

## 8. Using 2D Arrays
- **Slide Type**: Vertical Deep Dive
- **Question**: How do we handle 2D data?
- **Content**:
    -   **Declaration**: `DECLARE SeatingPlan : ARRAY[1:10, 1:5] OF STRING`
    -   **Assignment**: `SeatingPlan[1, 5] ← "Reserved"`
    -   **Nested Loops**:
        ```
        FOR Row ← 1 TO 10
            FOR Col ← 1 TO 5
                OUTPUT SeatingPlan[Row, Col]
            NEXT Col
        NEXT Row
        ```

## 9. Linear Search
- **Slide Type**: Process
- **Question**: How do we find something in an array?
- **Analogy**: Looking for a specific book on a shelf by checking one by one from left to right.
- **Steps**:
    1.  Start at the first element.
    2.  Check if it matches the target.
    3.  If yes, stop (found).
    4.  If no, move to the next.
    5.  Repeat until found or end of array.

## 10. Bubble Sort
- **Slide Type**: Process
- **Question**: How do we sort an array?
- **Analogy**: Bubbles rising to the top. Larger values "bubble" to the end.
- **Steps**:
    1.  Compare adjacent elements (1 & 2, 2 & 3...).
    2.  If they are in the wrong order, **swap** them.
    3.  Repeat the pass until no swaps are needed.
- **Visual**: Show a pass of `[5, 1, 4, 2, 8]`.

## 11. True or False
- **Slide Type**: Plenary
- **Questions**:
    1.  `MyArray[5]` accesses the 5th element (assuming 1-based). (True)
    2.  You can store Integers and Strings in the same array in pseudocode. (False - same data type)
    3.  A Linear Search requires the list to be sorted. (False - that's Binary Search)
