Topic B1: Computational Thinking

Complete Specification & Learning Guide

IB Computer Science

Thinking Like a Computer Scientist

Computational thinking is the problem-solving process that powers computer science. It involves breaking down complex problems, recognizing patterns, abstracting unnecessary details, and designing algorithms. In this topic, you will learn the core approaches to solving problems effectively before writing a single line of code.

What You Will Learn

In this section, we cover the fundamental problem-solving techniques:

  • Problem Specification: Defining problems clearly with inputs, outputs, and constraints.
  • Core Concepts: Abstraction, decomposition, pattern recognition, and algorithmic design.
  • Application: Using these concepts to solve real-world problems.
  • Flowcharts: Visualizing algorithms and control flow.

B1.1 Approaches to computational thinking

What you need to know:

  • B1.1.1 Construct a problem specification.
    • The specification of a problem may include a problem statement, constraints and limitations, objectives and goals, input specifications, output specifications, evaluation criteria.
  • B1.1.2 Describe the fundamental concepts of computational thinking.
    • Abstraction, algorithmic design, decomposition, pattern recognition.
  • B1.1.3 Explain how applying computational thinking to fundamental concepts is used to approach and solve problems in computer science.
    • Computational thinking does not necessarily involve programming—it is a toolkit of available techniques for problem-solving.
  • B1.1.4 Trace flowcharts for a range of programming algorithms.
    • Use of standard flowchart symbols to depict processes, decisions and flows of control.
    • Flowcharts for execution flow, to track changes in variables and to determine output.

Notes & Guidance

  • Real-world examples: Software development, data analysis, machine learning, database design, network security.
  • Standard flowchart symbols: Connector, Decision, Flowline, Input/Output, Process/Operation, Start/End.

Exam Expectations: What You Need to Do

For the IB Computer Science exam, you must be able to:

  • Define the Problem: Clearly articulate a problem statement, including its inputs, outputs, and constraints.
  • Apply Concepts: Demonstrate how abstraction, decomposition, and pattern recognition are used to simplify and solve complex problems.
  • Trace Algorithms: Follow the logic of a flowchart to determine the final state of variables or the output produced.
  • Design Solutions: Create flowcharts that represent logical solutions to specified problems.
