The Big Question

Building Blocks

How do we control the flow of a program to make decisions and repeat actions?

Three Constructs

Sequence, Selection, and Iteration (Loops).

Sequence

Instructions execute one after another in the exact order they are written.

Order matters! The swap algorithm fails if steps are out of order:

# Wrong order - both end up as 7!
a = b
temp = a
b = temp
# Correct order
temp = a
a = b
b = temp

Selection (IF Statements)

Execute different code based on a condition being true or false.

if purchase_value >= 100:
    discount = purchase_value * 0.2
else:
    discount = 0

Nested IF & ELIF

Use nested IFs or elif (else if) for multiple conditions.

if purchaseValue >= 100 and member:
    discount = 0.2
elif purchaseValue >= 100:
    discount = 0.1
else:
    discount = 0

Use and, or, not to combine conditions.

Use elif for multiple conditions.

Count-Controlled Loop (FOR)

Use when you know how many times to repeat in advance.

for i in range(1, 101):
    print(i)  # Prints 1 to 100

# With step of 2 (odd numbers)
for i in range(1, 100, 2):
    print(i)  # 1, 3, 5, ... 99

Pre-condition Loop (WHILE)

Use when the number of repetitions is unknown. Condition checked before each iteration. 

password = ""
while password != "secret":
    password = input("Enter password: ")
print("Access granted!")

Key: The loop might not run at all if the condition is already false.

Used for validation and verification.

Post-condition Loop

Condition checked after each iteration. Code runs at least once.

Python doesn't have do-while, but you can emulate it:

# Runs at least once
while True:
    answer = input("Enter 'quit' to exit: ")
    if answer == 'quit':
        break

Termination Condition

The condition that stops the loop.

Without it, you get an infinite loop! 

# Termination: when i >= 101
i = 1
while i < 101:
    print(i)
    i += 1  # Without this = infinite!

Functions & Procedures

Procedure

Does something, but no return value.

def greet(name):
    print("Hello", name)

Function

Returns a value.

def add(a, b):
    return a + b

result = add(3, 5)  # 8

Both promote maintainable code through reusability.

Variable Scope

Local Variable

Exists only inside the function/block where it's defined.

def test():
    x = 5  # local
test()
print(x)  # ERROR!

Global Variable

Exists throughout the entire program.

x = 5  # global
def test():
    global x
    x = 10
test()
print(x)  # 10

True or False?

Answer these misconception checks:

  1. "A WHILE loop always runs at least once."
  2. "A function must return a value."
  3. "The order of statements in a sequence doesn't matter."
  4. "Local variables can be accessed anywhere in the program."

Press Down to reveal answers

Question 1

"A WHILE loop always runs at least once."

Answer: False

WHILE is pre-condition - it may not run at all. Use while True: ... break to run at least once.

Question 2

"A function must return a value."

Answer: True

A function returns a value. A procedure does not.

Question 3

"The order of statements in a sequence doesn't matter."

Answer: False

Order is critical! Wrong order can cause logic errors.

Question 4

"Local variables can be accessed anywhere in the program."

Answer: False

Local = only in its block. Global = accessible everywhere.
