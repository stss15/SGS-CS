The Robot Chef Problem

A robot needs to make a sandwich. Order matters!

Algorithm A

  1. Get Bread
  2. Add Butter
  3. Add Ham
  4. Eat Sandwich

Algorithm B (Glitch)

  1. Eat Sandwich
  2. Get Bread
  3. Add Butter
  4. Add Ham

Crash! Cannot eat what doesn't exist.

Lesson: Sequence is fundamental.

Selection (IF)

Think of a locked door in a video game.

The Logic

if has_key == True:
    print("Door Opens")
else:
    print("Door Locked")

Visual Trace

1. Has Key? YES -> Enter IF block.

2. No Key? NO -> Jump to ELSE block.

Count-Controlled Loop (FOR)

Analogy: Running laps around a track.

You know exactly how many laps to run (e.g., 10).

# Run 5 laps
for lap in range(1, 6):
    print(f"Lap {lap}")

Output Predictor

  • Lap 1
  • Lap 2
  • ...
  • Lap 5

Pre-condition Loop (WHILE)

Analogy: A security Gatekeeper.

Must have the password before entering.

password = ""
while password != "secret":
    password = input("Guess: ")
print("Enter")

Danger: Infinite Loop if you never guess correctly!

Post-condition Loop

Analogy: Eating at a buffet.

You eat first, then check if you are full.

Always runs at least once.

Python Hack

Python doesn't have do...while. We fake it:

while True:
    eat_food()
    if is_full == True:
        break

Functions vs Procedures

Think of reusable Sub-routines as Recipes.

Procedure

Follows instructions, but gives you nothing back.

e.g., print_menu() displays text.

Function

Follows instructions AND returns a result (The Meal).

e.g., calculate_tax(price) returns a number.

Tracing a Function

Pass the ball (parameter) and catch the return.

def double_it(num):
    return num * 2

val = 10
result = double_it(val)
print(result)

Press Down to Trace

Step 1: Definition

Python learns what double_it means but does nothing yet.

Step 2: The Call

We call double_it(10).

Inside the function, num becomes 10.

Step 3: The Return

num * 2 is 20.

The function throws 20 back to result.

Variable Scope

Analogy: Tinted Windows on a Car.

Global (Outside Car)

Can see inside? NO.

Cannot access variables created inside a function (Local).

Local (Inside Car)

Can see outside? YES.

Function can read Global variables.

Predict the Scope

score = 10  # Global

def update():
    new_score = 20  # Local
    print(score)

update()
print(new_score)

Output Line 1: 10 (Can see global)

Output Line 2: NameError! (Cannot see local 'new_score')
