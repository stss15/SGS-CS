Topic B2: Programming | Arrays, Stacks & Queues

The Big Question

Storing Data

How do we store 1000 student grades without creating 1000 separate variables?

Data Structures

We use organized collections like arrays, stacks, and queues.

Static vs Dynamic Data Structures

Static Data Structure

  • Fixed size (set at creation)
  • Contiguous memory locations
  • Fast direct access via index
  • Example: Arrays

Dynamic Data Structure

  • Can grow/shrink at runtime
  • Non-contiguous memory
  • Sequential access (slower)
  • Example: Linked Lists

Arrays (1D)

An array stores multiple values of the same type under one name, accessed by index. 

grades = [75, 64, 95, 50, 34]
print(grades[2])  # 95
print(grades[0])  # 75 (first element)
print(grades[-1]) # 34 (last element)

Note: Indexing starts at 0. Element at index 2 is the 3rd element. Use -1 for last element.

Two-Dimensional Arrays

A 2D array is like a table with rows and columns. Access elements with two indices: array[row][col] 

# Create 3x3 grid initialized to 0
nums = [[0]*3 for _ in range(3)]
nums[1][2] = 5  # row 1, col 2
print(nums[1][2])  # 5

Parallel Arrays

Multiple arrays where elements at the same index are related.

names = ["Alice", "Bob", "Carol"]
ages  = [17, 18, 16]
# names[1] and ages[1] both refer to Bob

Useful for storing related data (like a simple database record) without using objects.

What is a Stack?

The Concept

Think of a stack of plates. You add to the top and remove from the top.

LIFO: Last In, First Out

Operations

  • Push: Add to top
  • Pop: Remove from top
  • Peek: View top element
  • isEmpty / isFull

How do stack operations work?

Press Down to see step-by-step.

Push(5): Add 5 to empty stack.

Stack: [5] | Top: 0

Push(12): Add 12 on top.

Stack: [5, 12] | Top: 1

Push(9): Add 9 on top.

Stack: [5, 12, 9] | Top: 2

Pop(): Remove 9 (top element).

Stack: [5, 12] | Top: 1

Peek(): Returns 12 (current top) without removing it.

Uses of Stacks

  • Undo feature: Actions pushed, undo pops.
  • Browser back button: Pages pushed on visit.
  • Interrupt handling: CPU state saved.
  • Expression evaluation: RPN calculations.
  • Recursion: Return addresses stored.

What is a Queue?

The Concept

Think of a line at a checkout. First person in line is served first.

FIFO: First In, First Out

Operations

  • Enqueue: Add to rear
  • Dequeue: Remove from front
  • Front: View front element
  • isEmpty / isFull

Uses of Queues

  • Printer queue: Jobs printed in order received.
  • Keyboard buffer: Characters displayed in order typed.
  • Simulations: Checkout lines, car washes, etc.
  • Task scheduling: Operating system job queues.

True or False?

Answer these misconception checks:

  1. "A stack uses the FIFO principle."
  2. "In a 2D array, you access elements using two indices."
  3. "Dynamic data structures have a fixed size."
  4. "Enqueue adds an element to the front of a queue."

Press Down to reveal answers

Question 1

"A stack uses the FIFO principle."

Answer: False

Stacks use LIFO (Last In First Out). Queues use FIFO.

Question 2

"In a 2D array, you access elements using two indices."

Answer: True

Correct. First index for row, second for column.

Question 3

"Dynamic data structures have a fixed size."

Answer: False

Dynamic structures can grow/shrink. Static structures have fixed size.

Question 4

"Enqueue adds an element to the front of a queue."

Answer: False

Enqueue adds to the rear. Dequeue removes from the front.
