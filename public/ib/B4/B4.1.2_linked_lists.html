Linked Lists

Quick Reference Study Guide & Revision Notes

Linked List Structure

A linked list is a linear data structure where elements are stored in nodes. Unlike arrays, linked lists do not store data contiguously in memory.

Key Components

Component Description
Node A basic unit containing data and a reference (pointer) to the next node.
Head The first node in the linked list. Used as the starting point for traversal.
Tail The last node in the list. Its pointer points to None (null).
Pointer A variable that stores the memory address of the next node.

Visual Representation

head → A → B → C → None
Analogy

Think of a linked list like a chain of elephants. Each elephant (node) holds the tail of the next elephant (pointer). The first elephant is the head, and the last elephant's trunk points to nothing (null).

Advantages & Disadvantages

✅ Advantages

  • Dynamic Size: Can grow and shrink at runtime (unlike static arrays).
  • Efficient Insertion/Deletion: No need to shift elements like in an array.
  • Memory Efficiency: No pre-allocated memory; nodes are created as needed.

❌ Disadvantages

  • Sequential Access: Must traverse from head; no direct index access like arrays.
  • Extra Memory: Each node requires additional memory for the pointer.
  • No Binary Search: Linear search only (O(n) complexity).
Exam Tip

Heap Exhaustion: If a linked list grows too large, it can exhaust available heap memory, causing the program to crash.

Types of Linked Lists

Singly Linked List

Each node has one pointer to the next node. The tail points to None.

head → A → B → C → None

Doubly Linked List

Each node has two pointers: one to the next node and one to the previous node.

None ← A ⇄ B ⇄ C → None

Circular Linked List

The tail node points back to the head instead of None.

head → A → B → C → (back to head)
Common Mistake

Don't forget to label the head and ensure the tail node points to None (or back to head in circular lists) in your diagrams.

Linked List Operations

Traversal / Search

Start at the head and follow pointers until you find the target or reach None.

  1. Start with the head pointer to access the first node.
  2. Follow the pointer within the node to move to the next node.
  3. Repeat until you find the target or reach None.

Insertion

Position Steps
At Beginning 1. Create new node. 2. Point new node to current head. 3. Update head to new node.
At End 1. Create new node. 2. Traverse to tail. 3. Point tail to new node.
In Middle 1. Create new node. 2. Find target node. 3. Point new node to target's next. 4. Point target to new node.

Deletion

Position Steps
First Node 1. Check if list is empty. 2. Update head to point to the second node.
Middle Node 1. Traverse to find node (keep track of previous). 2. Point previous node to the found node's next.
Last Node 1. Traverse to find tail (keep track of previous). 2. Point previous node to None.
Common Mistake

When inserting in the middle, always update the new node's pointer first before changing the previous node's pointer. Otherwise, you'll lose the reference to the rest of the list!

Python Implementation

Node Class

class ListNode:
    """A node in a singly linked list."""
    def __init__(self, data=0):
        self.data = data      # Store the data
        self.next = None      # Pointer to next node

LinkedList Class - Setup & Traversal

class LinkedList:
    """A singly linked list."""
    def __init__(self):
        self.head = None      # Initialize head pointer

    def print_list(self):
        """Traverse and print all nodes."""
        current = self.head
        while current is not None:
            print(current.data, end=" -> ")
            current = current.next
        print("None")

Insert at Beginning

def insert_at_beginning(self, data):
    """Insert a new node at the start of the list."""
    new_node = ListNode(data)      # Create new node
    new_node.next = self.head      # Point to current head
    self.head = new_node           # Update head pointer

Insert at End

def insert_at_end(self, data):
    """Insert a new node at the end of the list."""
    new_node = ListNode(data)
    
    if self.head is None:          # If list is empty
        self.head = new_node
    else:
        current = self.head
        while current.next is not None:  # Traverse to last node
            current = current.next
        current.next = new_node    # Point last node to new node

Insert After a Value

def insert_after_value(self, target_value, data):
    """Insert a new node after the first node with target_value."""
    current = self.head
    
    while current is not None:
        if current.data == target_value:
            new_node = ListNode(data)
            new_node.next = current.next  # Point to next node
            current.next = new_node       # Insert new node
            return
        current = current.next
    
    print(f"Node with data {target_value} not found.")

Delete a Node by Value

def delete_node(self, target_value):
    """Delete the first node with the given value."""
    current = self.head
    previous = None
    
    while current is not None:
        if current.data == target_value:
            if previous is None:      # Deleting head
                self.head = current.next
            else:
                previous.next = current.next
            return
        previous = current
        current = current.next
    
    print(f"Node with data {target_value} not found.")

Complete Example

# Create a linked list
my_list = LinkedList()

# Insert nodes
my_list.insert_at_beginning(10)
my_list.insert_at_end(20)
my_list.insert_at_end(30)
my_list.insert_after_value(20, 25)

my_list.print_list()
# Output: 10 -> 20 -> 25 -> 30 -> None

# Delete a node
my_list.delete_node(25)
my_list.print_list()
# Output: 10 -> 20 -> 30 -> None
Keyword Glossary
Keyword Definition
Node A basic unit of a data structure (e.g., a linked list) which contains data and typically links to other nodes.
Pointer A variable that stores the memory address of another variable, used to reference or access data stored in memory.
Head The first node in a linked list; the entry point for traversal.
Tail The last node in a linked list; its pointer is None.
Traversal The process of visiting each node in a data structure, typically from head to tail.
Dynamic Data Structure A data structure that can grow or shrink in size during program execution.
Heap Space A region of dynamically allocated memory managed by the OS where programs store data structures that grow at runtime.
Garbage Collection An automatic memory management process that reclaims memory no longer in use by the program.
