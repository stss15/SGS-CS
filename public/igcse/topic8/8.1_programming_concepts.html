The Big Question

How does a computer remember things?

When you play a game, how does it know your score?

When you type your name, where does it go?

Computers use variables – named storage containers that hold data.

Learn about variables

Variables & Constants

Variable

A named storage location whose value can change.

Like a whiteboard – you can erase and rewrite.

Example: Score, Name

Constant

A named storage location whose value never changes.

Like carving in stone – fixed forever.

Example: Pi = 3.142

How to declare them

Declaring & Assigning

Declaration

Tell the computer what name and type.

DECLARE Score : INTEGER
DECLARE Name : STRING
CONSTANT Pi = 3.142

Assignment

Put a value into the variable.

Score ← 10
Name ← "Ali"

Use ← (not =) for assignment!

The Three Programming Constructs

Every program is built from just three things:

Sequence

Lines run one after another, in order.

Selection

Make decisions (IF/ELSE).

Iteration

Repeat code (loops).

Sequence

Code runs line by line, top to bottom.

OUTPUT "What is your name?"
INPUT Name
OUTPUT "Hello, ", Name

Line 1 runs first, then Line 2, then Line 3.

Simple but essential!

Selection (IF)

What if we need to make a decision?

Analogy: A fork in the road – which path do you take?

IF statements

IF...THEN...ELSE

IF condition THEN
    statements
ELSE
    statements
ENDIF

Example: Pass or Fail

INPUT Mark
IF Mark >= 50 THEN
    OUTPUT "Pass"
ELSE
    OUTPUT "Fail"
ENDIF

Key: Always close with ENDIF

Nested IF

Sometimes you need multiple conditions:

INPUT Mark
IF Mark >= 70 THEN
    OUTPUT "Distinction"
ELSE
    IF Mark >= 50 THEN
        OUTPUT "Pass"
    ELSE
        OUTPUT "Fail"
    ENDIF
ENDIF

CASE Statements

When you have many options, use CASE instead of nested IF:

INPUT Day
CASE OF Day
    "Monday": OUTPUT "Start of week"
    "Friday": OUTPUT "Nearly weekend!"
    "Saturday", "Sunday": OUTPUT "Weekend!"
    OTHERWISE: OUTPUT "Mid-week"
ENDCASE

Key: More readable than many IF statements. Use OTHERWISE for default.

Iteration (Loops)

What if we need to repeat something?

Imagine writing "Hello" 100 times without loops...

Loops let us repeat code automatically!

Three types of loops

FOR Loop

Use when: You know exactly how many times to repeat.

FOR Count ← 1 TO 5
    OUTPUT Count
NEXT Count

Output: 1, 2, 3, 4, 5

Repeats 5 times, Count increases by 1 each time.

WHILE Loop

Use when: You repeat while a condition is true.

Checks condition before each loop.

Count ← 1
WHILE Count <= 5
    OUTPUT Count
    Count ← Count + 1
ENDWHILE

Key: Might run zero times if condition is false from the start!

REPEAT...UNTIL Loop

Use when: You repeat until a condition becomes true.

Checks condition after each loop.

REPEAT
    OUTPUT "Enter password:"
    INPUT Password
UNTIL Password = "secret123"

Key: Always runs at least once!

Choosing the Right Loop

Loop When to Use Minimum Runs
FOR Know exact number of times 0 (if range is invalid)
WHILE Condition checked first 0
REPEAT Must run at least once 1

Key Takeaways

  1. Variables store data that can change; Constants are fixed
  2. 5 data types: INTEGER, REAL, CHAR, STRING, BOOLEAN
  3. 3 constructs: Sequence, Selection, Iteration
  4. Selection: IF/ELSE, CASE
  5. Iteration: FOR (known count), WHILE (pre-check), REPEAT (post-check)

True or False?

Check Your Understanding

  1. A constant can change its value during the program.
  2. A WHILE loop always runs at least once.
  3. BOOLEAN can only be TRUE or FALSE.

Press Down to reveal answers

Question 1

A constant can change its value during the program.

Answer: FALSE

A constant is fixed and cannot change. That's the whole point!

Question 2

A WHILE loop always runs at least once.

Answer: FALSE

WHILE checks the condition before running. If false from the start, it runs zero times. REPEAT runs at least once.

Question 3

BOOLEAN can only be TRUE or FALSE.

Answer: TRUE

BOOLEAN data type has exactly two possible values: TRUE and FALSE. Nothing else!
