Arrays to the rescue

What is an Array?

Definition: A data structure containing multiple elements of the same type, accessed using a single name and an index.

Analogy: Think of a row of lockers.

Each locker has the same name ("Locker") but a different number (1, 2, 3...)

[1]

Ali

[2]

[3]

John

[4]

[5]

Array: Names

1D Arrays

A single list of items

How to create and use them

Declaring a 1D Array

DECLARE Names : ARRAY[1:5] OF STRING

Breakdown

Names - name of array

[1:5] - indexes 1 to 5 (5 slots)

STRING - type of data

More Examples

DECLARE Marks : ARRAY[1:30] OF INTEGER
DECLARE Prices : ARRAY[1:100] OF REAL
DECLARE Passed : ARRAY[1:10] OF BOOLEAN

Accessing Array Elements

Use the index (position number) in square brackets:

Writing (Assignment)

Names[1] ← "Ali"
Names[2] ← "Sara"
Names[3] ← "John"

Reading (Access)

OUTPUT Names[1]
OUTPUT Names[3]

Outputs: Ali, John

Key: The index tells you which "slot" to look at.

Looping Through Arrays

This is where arrays become powerful!

Use a FOR loop to process every element:

FOR i ← 1 TO 5
    OUTPUT Names[i]
NEXT i

Magic: i changes from 1 to 5, so we access Names[1], Names[2], Names[3], Names[4], Names[5]!

Example: Totalling Marks

DECLARE Marks : ARRAY[1:5] OF INTEGER
DECLARE Total : INTEGER

Total ← 0
FOR i ← 1 TO 5
    Total ← Total + Marks[i]
NEXT i
OUTPUT "Total: ", Total

This adds up all 5 marks in the array using a loop!

Example: Input into Array

DECLARE Names : ARRAY[1:30] OF STRING

FOR i ← 1 TO 30
    OUTPUT "Enter name ", i
    INPUT Names[i]
NEXT i

This fills the entire array with 30 names from user input!

2D Arrays

What if we need a table of data?

Analogy: Think of a spreadsheet with rows and columns.

Or cinema seats – Row 3, Seat 5.

Two dimensions: row and column

Visualising 2D Arrays

Students (rows) × Subjects (columns)

Col 1 Col 2 Col 3
Row 1 85 72 91
Row 2 63 88 77
Row 3 95 81 69

Marks[2, 2] = 88 (Row 2, Column 2)

Declaring 2D Arrays

DECLARE Marks : ARRAY[1:3, 1:3] OF INTEGER

Breakdown

[1:3, 1:3]

First range = rows (1 to 3)

Second range = columns (1 to 3)

Total: 3 × 3 = 9 elements

Accessing

Marks[1, 1] ← 85
Marks[2, 3] ← 77
OUTPUT Marks[3, 2]

[row, column]

Looping Through 2D Arrays

Use nested loops – one for rows, one for columns:

FOR Row ← 1 TO 3
    FOR Col ← 1 TO 3
        OUTPUT Marks[Row, Col]
    NEXT Col
NEXT Row

How it works:

Row 1: output [1,1], [1,2], [1,3]

Row 2: output [2,1], [2,2], [2,3]

Row 3: output [3,1], [3,2], [3,3]

Linear Search in Arrays

INPUT SearchName
Found ← FALSE
FOR i ← 1 TO 30
    IF Names[i] = SearchName THEN
        Found ← TRUE
        OUTPUT "Found at position ", i
    ENDIF
NEXT i
IF Found = FALSE THEN
    OUTPUT "Not found"
ENDIF

Checks each element one by one until found (or reaches the end).

Key Takeaways

  1. An array stores multiple values of the same type
  2. Access elements using an index: Names[3]
  3. 1D arrays = single list, 2D arrays = table (rows & columns)
  4. Use FOR loops to process all elements
  5. 2D arrays use nested loops: [Row, Col]

True or False?

Check Your Understanding

  1. An array can store different data types in the same structure.
  2. To access an element, you use its index.
  3. 2D arrays need nested loops to process all elements.

Press Down to reveal answers

Question 1

An array can store different data types in the same structure.

Answer: FALSE

All elements in an array must be the same data type. You can't mix strings and integers!

Question 2

To access an element, you use its index.

Answer: TRUE

The index is the position number that identifies which element you want.

Question 3

2D arrays need nested loops to process all elements.

Answer: TRUE

You need one loop for rows and another (inside) for columns to visit every element.
