Bringing Code to Life

Topic 7 taught you how to design algorithms; Topic 8 is about building them. This chapter covers the practical skills needed to write, test, and refine working computer programs. You will learn to control data flow with loops and logic, organise code with functions and procedures, and handle data persistence using files.

What You Will Learn

This topic covers the essential syntax and structures for programming:

  • Programming Concepts: Variables, constants, data types, input/output, and operators (Arithmetic, Logical, Boolean).
  • Control Flow: Sequence, Selection (IF, CASE), and Iteration (FOR, REPEAT, WHILE).
  • Data Handling: String manipulation (Length, Substring, Upper/Lower) and Library Routines (Random, Round).
  • Subroutines: Creating and using Procedures and Functions, including parameters and local/global variables.
  • Arrays: Using 1D and 2D arrays to store lists and tables of data.
  • File Handling: Reading from and writing to external text files to save data permanently.

8.1 Programming Concepts

What you need to know:

  • Basics:
    • Declare and use variables and constants.
    • Understand and use basic data types (Integer, Real, Char, String, Boolean).
    • Understand and use Input and Output statements.
  • Constructs:
    • Sequence: Instructions executed one after another.
    • Selection: IF statements and CASE statements (including nesting).
    • Iteration: Count-controlled loops (FOR), Pre-condition loops (WHILE), and Post-condition loops (REPEAT).
  • Operations:
    • Totalling & Counting: Standard algorithms for summing and tallying.
    • String Handling: Length, Substring, Upper, Lower.
    • Operators: Arithmetic (+, -, *, /, ^, MOD, DIV), Relational (=, <, <=,>, >=, <>), and Logical (AND, OR, NOT).
  • Subroutines:
    • Define and use Procedures and Functions (with or without parameters).
    • Understand the difference between local and global variables.
    • Use Library Routines (MOD, DIV, ROUND, RANDOM).
  • Maintainability: Understand how to create a maintainable program (meaningful identifiers, comments, indentation).

Notes & Guidance

  • String Indexing: The first character of a string can be position 0 or 1 depending on the language. In pseudocode, we often assume 1-based indexing unless specified.
  • Nested Statements: You need to be comfortable nesting IFs inside Loops, Loops inside Loops, etc. (Max depth usually 3 levels).
  • Procedures vs Functions: A Procedure performs a task but does not return a value. A Function performs a task and must return a value.
  • Operators:
    • MOD returns the remainder.
    • DIV returns the integer quotient (e.g., 7 DIV 2 = 3).
    • <> means "Not Equal To".

8.2 Arrays

What you need to know:

  • Declaration: Declare and use one-dimensional (1D) and two-dimensional (2D) arrays.
  • Usage: Understand the use of arrays to store multiple items of data under a single identifier.
  • Iteration: Write values into, and read values from, an array using iteration (loops).

Notes & Guidance

  • Indexing: Use variables as indexes (e.g., MyArray[Index]). The first index can be 0 or 1.
  • Nested Iteration: To traverse a 2D array (like a grid or table), you use a nested loop (an outer loop for rows, an inner loop for columns).
  • Example: StudentNames[1] = "Ali" or Grid[2,3] = 'X'.

8.3 File Handling

What you need to know:

  • Purpose: Understand the purpose of storing data in a file (permanent storage that persists after the program closes).
  • Operations: Open, close, and use a file for reading and writing.

Notes & Guidance

  • Modes: Files must be opened in a specific mode: READ or WRITE.
  • Writing: WRITEFILE creates a new file or overwrites an existing one.
  • Reading: READFILE reads data from an existing file (usually line by line).
  • Closing: Always CLOSEFILE when finished to prevent data corruption.

Exam Expectations: What You Need to Do

For the Cambridge IGCSE (0478) exam, you must be able to:

  • Write Code: Write small snippets of pseudocode or program code (Python/VB/Java) to solve specific problems (e.g., "Write a loop to output the first 10 square numbers").
  • Debug: Identify logic errors or syntax errors in a given piece of code and correct them.
  • Explain Concepts: Explain why you would use a constant instead of a variable, or why a loop is more efficient than repeating code.
  • Trace Arrays: Manually trace the values in an array as a loop updates them (e.g., performing a Bubble Sort pass).
  • Handle Files: Write the correct sequence of commands to open a file, write a string to it, and close it safely.
