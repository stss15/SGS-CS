Deep Dive: Low-Level Languages

Deep Dive: Low-Level Languages

Languages closer to the hardware. Hard for humans, fast for machines.

1. Machine Code

Binary instructions (1s and 0s) executed directly by the CPU.

1010 0101 1100

2. Assembly Language

Uses Mnemonics (short codes) to represent machine code.

LDA First
ADD Second
STO Sum

Comparison

High-Level
  • Easy to debug
  • Portable
  • Slower execution
Low-Level
  • Fast execution
  • Hardware control
  • Hard to learn
Why use Assembly?

To write device drivers or code that needs to run extremely fast (e.g. games engines).

Translators

Software that converts code into machine code.

Assembler

Assembly → Machine Code.

Compiler

HLL → Machine Code (All at once).

Interpreter

HLL → Machine Code (Line by line).

Deep Dive: Compiler vs Interpreter

Deep Dive: Compiler vs Interpreter

Crucial for the exam!

Compiler

  • Translates whole program at once.
  • Creates an Executable (.exe) file.
  • Reports errors at the end.
  • Use: Distributing finished software.

Interpreter

  • Translates one line at a time.
  • Does NOT create an executable.
  • Stops at the first error.
  • Use: Developing/Testing code.

Question 1

"A compiler translates code line by line while the program runs."

Answer: False

A compiler translates the whole program first, producing an executable. An interpreter runs line-by-line.

Question 2

"An assembler converts assembly language into machine code."

Answer: True

Assemblers translate mnemonics directly into the binary opcodes the CPU executes.

Question 3

"Prettyprinting is a debugger feature that executes code step-by-step."

Answer: False

Prettyprinting is formatting/syntax colouring. Stepping through code is done with the debugger.
