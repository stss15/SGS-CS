The Physical Engine

Hardware is the physical machinery that makes computing possible. While software provides the instructions, hardware does the heavy lifting. In this massive unit, we explore everything from the microscopic transistors inside a CPU to the sensors, printers, and storage devices that allow computers to interact with the physical world.

What You Will Learn

This is a large topic covering the physical components of computer systems:

  • Computer Architecture: The Von Neumann model, the Fetch-Decode-Execute cycle, and what makes a CPU fast (cores, cache, clock).
  • Input & Output: The vast array of devices used to get data in (sensors, scanners) and out (screens, 3D printers).
  • Data Storage: Where data lives when we aren't using it (HDD, SSD, Optical) and where it lives when we are (RAM, ROM).
  • Network Hardware: The specialized kit (NICs, Routers) that connects our isolated machines to the world.

3.1 Computer Architecture

What you need to know:

  • The CPU:
    • (a) Understand the role of the Central Processing Unit (CPU) in a computer.
    • (b) Understand what is meant by a microprocessor.
  • Von Neumann Architecture:
    • (a) Understand the purpose of components in a CPU (Von Neumann architecture):
      • Units: Arithmetic Logic Unit (ALU) and Control Unit (CU).
      • Registers: Program Counter (PC), Memory Address Register (MAR), Memory Data Register (MDR), Current Instruction Register (CIR), and Accumulator (ACC).
      • Buses: Address bus, Data bus, and Control bus.
    • (b) Describe the process of the Fetch-Decode-Execute (FDE) cycle, including the role of each component in the process.
  • Performance Factors: Understand what is meant by a core, cache, and clock in a CPU and explain how they can affect the performance of a CPU.
  • Instruction Set: Understand the purpose and use of an instruction set for a CPU.
  • Embedded Systems: Describe the purpose and characteristics of an embedded system and identify devices in which they are commonly used.

Notes & Guidance

  • The CPU's Role: Processes instructions and data input into the computer so results can be output.
  • Microprocessor: A type of integrated circuit on a single chip.
  • The FDE Cycle Details:
    • How instructions/data are fetched from RAM into the CPU.
    • How they are processed using each component.
    • How they are then executed.
    • You must know how data/addresses are stored in specific registers and transmitted via buses.
  • Performance: Increasing the number of cores, size of the cache, or speed of the clock can improve performance.
  • Instruction Set: A list of all commands (machine code) that can be processed by a specific CPU.
  • Embedded Systems:
    • Used to perform dedicated functions (unlike a general-purpose PC).
    • Examples: Domestic appliances, cars, security systems, lighting systems, vending machines.

3.2 Input and Output Devices

What you need to know:

  • Input Devices: Understand what is meant by an input device and why it is required. Identify and describe:
    • Barcode scanner, QR code scanner.
    • Digital camera, Keyboard, Microphone, Optical mouse.
    • Touch screen (Resistive, Capacitive, Infra-red).
    • 2D and 3D scanners.
  • Output Devices: Understand what is meant by an output device and why it is required. Identify and describe:
    • Actuator.
    • DLP (Digital Light Processing) projector, LCD projector.
    • Inkjet printer, Laser printer, 3D printer.
    • LED screen, LCD screen.
    • Speaker.
  • Sensors:
    • (a) Understand what is meant by a sensor and the purposes of sensors.
    • (b) Identify the type of data captured by each sensor and select the most suitable sensor for a given context.

Notes & Guidance

  • Sensors List (Limited to): Acoustic, Accelerometer, Flow, Gas, Humidity, Infra-red, Level, Light, Magnetic field, Moisture, pH, Pressure, Proximity, Temperature.
  • Touch Screens: You need to know the specific technologies: resistive (pressure), capacitive (electrical charge of finger), and infra-red (grid of beams).
  • 3D Printers: Create physical objects by building layer upon layer.
  • Actuators: Motors or devices that create physical movement (used in control systems).

3.3 Data Storage

What you need to know:

  • Primary Storage:
    • Understand what is meant by primary storage (directly accessed by CPU).
    • Roles of RAM (Random Access Memory) and ROM (Read Only Memory).
    • Why a computer needs both RAM and ROM, and the difference between them.
  • Secondary Storage:
    • Understand what is meant by secondary storage (not directly accessed by CPU; permanent storage).
    • Describe the operation of Magnetic, Optical, and Solid-state (flash memory) storage and give examples.
  • Virtual Memory: Describe what is meant by virtual memory, how it is created and used, and why it is necessary.
  • Cloud Storage:
    • Understand what is meant by cloud storage.
    • Explain the advantages and disadvantages of storing data on the cloud vs locally.

Notes & Guidance

  • Magnetic Storage: Uses platters divided into tracks and sectors. Data is read/written using electromagnets (e.g., HDD).
  • Optical Storage: Uses lasers to create and read pits and lands (e.g., CD, DVD, Blu-ray).
  • Solid-state Storage: Uses NAND or NOR technology. Transistors are used as control gates and floating gates (e.g., SSD, SD card, USB drive).
  • Virtual Memory: Pages of data are transferred between RAM and virtual memory (on the HDD/SSD) when RAM is full.
  • Cloud Storage: Accessed remotely. Requires physical servers and storage at the provider's location.

3.4 Network Hardware

What you need to know:

  • NIC: Understand that a computer needs a Network Interface Card (NIC) to access a network.
  • MAC Address:
    • Understand what is meant by, and the purpose of, a Media Access Control (MAC) address.
    • Understand its structure (Manufacturer code + Serial code).
  • IP Address:
    • Understand what is meant by, and the purpose of, an Internet Protocol (IP) address.
    • Understand there are different types (IPv4 and IPv6; Static and Dynamic).
  • Router: Describe the role of a router in a network.

Notes & Guidance

  • MAC Address: Given at point of manufacture. Usually written in hexadecimal.
  • IP Address: Allocated by the network. IPv4 (32-bit, dot-decimal) vs IPv6 (128-bit, hex).
  • Router Roles:
    • Sends data to a specific destination on a network.
    • Can assign IP addresses.
    • Can connect a local network (LAN) to the internet (WAN).

Exam Expectations: What You Need to Do

For the Cambridge IGCSE (0478) exam, you must be able to:

  • Describe Mechanics: Explain exactly how an HDD reads data (platters, electromagnets) versus how an Optical disc works (lasers, pits/lands).
  • Trace the Cycle: Describe the Fetch-Decode-Execute cycle step-by-step, mentioning the specific registers (MAR, MDR, PC, etc.) involved at each stage.
  • Select Devices: Given a scenario (e.g., "a blind person using a computer"), select the appropriate input/output devices and justify your choice.
  • Differentiate Addresses: Explain the difference between a MAC address (physical, permanent) and an IP address (logical, changeable).
