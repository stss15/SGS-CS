The Big Question

The Desk

Where you do your work (Fast, Temporary)

Quick access, but limited space

The Cabinet

Where you store your files (Slow, Permanent)

Lots of space, but takes time to access

Key Idea: We need both. RAM (Desk) for speed, HDD/SSD (Cabinet) for storage. Different jobs, different tools!

Storage Hierarchy

Primary Memory

What memory does the CPU access directly?

Definition: Directly accessible by the CPU.

Like the CPU's immediate workspace - no waiting!

RAM

Random Access Memory

Volatile (Lost when power off)

Read/Write

Stores running programs/data

Like your desk - temporary workspace.

ROM

Read Only Memory

Non-Volatile (Kept when power off)

Read Only

Stores BIOS/Bootstrap

Like a permanent instruction manual.

Deep Dive: DRAM vs SRAM

RAM vs ROM

DRAM vs SRAM

Why are there two types of RAM?

DRAM (Dynamic)

Needs constant refreshing (data leaks away!)

Cheaper, Higher density

Used for Main RAM

Like a bucket with a leak - needs constant refilling.

SRAM (Static)

No refresh needed (uses flip-flops - stable!)

Faster, Expensive

Used for Cache

Like a sealed container - holds data without leaking.

Secondary Storage

Where do we store data permanently?

Characteristics: Non-volatile, Slower, Larger Capacity

Like a filing cabinet - permanent storage, but takes time to access.

Magnetic

HDD (Hard Disk Drive)

Platters, Read/Write heads

Like a record player - spinning disks.

Optical

CD, DVD, Blu-ray

Lasers, Pits & Lands

Like a CD player - laser reads the surface.

Solid State

SSD, USB

Flash Memory (Electrons)

Like a memory card - no moving parts!

Optical Media

Which to Choose?

HDD, SSD, or Optical? It depends on what you need!

HDD (Hard Disk Drive)

Pros: Cheap, Huge Capacity

Cons: Fragile (Moving parts), Slower

Best for: Large files, backups, budget builds.

SSD (Solid State Drive)

Pros: Very Fast, Durable, Silent

Cons: Expensive (per GB)

Best for: Operating system, games, speed-critical tasks.

Optical (Blu-ray/DVD)

Pros: Portable, Cheap distribution

Cons: Slow, Can get scratched

Best for: Movies, software distribution, archives.

Virtual Memory

What happens when RAM runs out?

The Problem

RAM is full. The computer is about to crash.

Like your desk overflowing with papers!

The Solution

Use part of the HDD/SSD as "fake RAM"

Move inactive data (pages) out of RAM to the HDD

Like moving old papers to a filing cabinet to make room on your desk.

Warning: Disk Thrashing. If you swap too much, the computer becomes incredibly slow.

Like constantly moving papers back and forth - very inefficient!

Cloud Storage

What if we store data somewhere else entirely?

Definition: Storing data on remote servers (The Internet)

Like a filing cabinet in someone else's building, accessible via the internet.

Advantages

  • Access from anywhere: Any device, any location
  • Automatic backups: Your data is safe
  • Easy to share: Send links instead of files

Like having your files follow you everywhere.

Disadvantages

  • Need internet: No connection = no access
  • Security risks: Hacking, data breaches
  • Monthly costs: Subscription fees

Like trusting someone else with your important documents.

True or False?

Misconception Check

  1. "RAM keeps its data when power is off."
  2. "SSD has moving parts."
  3. "Virtual memory is faster than RAM."
  4. "Blu-ray uses a red laser."

Press Down to reveal answers

Question 1

"RAM keeps its data when power is off."

Answer: False

RAM is Volatile. ROM is Non-Volatile.

Question 2

"SSD has moving parts."

Answer: False

Solid State means NO moving parts. HDD has moving parts.

Question 3

"Virtual memory is faster than RAM."

Answer: False

It uses the HDD/SSD, which is much slower than RAM.

Question 4

"Blu-ray uses a red laser."

Answer: False

It uses a Blue/Violet laser (shorter wavelength = more data). CD/DVD use Red.
