Von Neumann Architecture

How should a computer be organized?

The Stored Program Concept

Programs and data are stored in the SAME memory.

Instructions are fetched and executed sequentially (one after another).

Revolutionary idea: The computer can modify its own instructions!

Fetch-Decode-Execute Cycle

How does the CPU actually process instructions?

Analogy: Like a chef following a recipe. Get the instruction (Fetch), understand it (Decode), do it (Execute).

1. Fetch

Get the next instruction from memory (RAM)

Like getting the next step from the recipe book.

2. Decode

The Control Unit figures out what the instruction means

Like reading and understanding what "add 2 eggs" means.

3. Execute

Carry out the instruction (e.g., Add numbers, Load data)

Like actually adding the eggs to the bowl.

The Register Dance

Step-by-step: How do registers work together?

  1. PC (Program Counter) holds the address of the next instruction.
  2. Address is copied to MAR (Memory Address Register) - "Where to look"
  3. Instruction at that address is fetched to MDR (Memory Data Register) - "What we found"
  4. Instruction copied to CIR (Current Instruction Register) - "What we're working on"
  5. PC is incremented (PC + 1) - "Get ready for next instruction"
  6. Instruction in CIR is Decoded - "What does this mean?"
  7. Instruction is Executed (result might go to ACC (Accumulator)) - "Do it!"

This happens billions of times per second!

CPU Performance

What makes one CPU faster than another?

Clock Speed

Measured in GHz (billions of cycles per second)

More = Faster

Like RPM in a car - more revolutions = faster processing.

Cores

Multiple CPUs on one chip

Dual Core, Quad Core, Octa Core

More = Better multitasking

Like having multiple workers instead of one.

Cache

Super fast memory ON the CPU

Stores frequently used data

More = Less waiting for RAM

Like a desk drawer vs filing cabinet - instant access!

Deep Dive: Pros & Cons

Embedded Systems

Embedded Systems: The Trade-off

Advantages

  • Reliable: Simple software (does one job well)
  • Cheap: Mass produced in millions
  • Low power: Designed to be efficient
  • Small size: Fits inside the device

Perfect for dedicated tasks!

Disadvantages

  • Hard to upgrade: Built into the device
  • Hard to fix: Often need to replace entire device
  • Limited resources: Small memory/slow speed (but enough for the job)

Not designed for flexibility.

Question 1

"Data and programs are stored in separate memories in Von Neumann architecture."

Answer: False

They share the SAME memory. This is the key feature.

Question 2

"The ALU controls the flow of data."

Answer: False

The Control Unit (CU) controls the flow. The ALU does the maths.

Question 3

"An embedded system is a general-purpose computer."

Answer: False

It has a dedicated function (e.g., washing clothes).

Question 4

"More cores always means double the speed."

Answer: False

Communication between cores takes time, and not all software is written to use multiple cores.
