# 9.3 Building Databases (Extension) Slide Plan

## 1. Title Slide
- **Slide Type**: Title
- **Title**: 9.3 Building Databases (Extension)
- **Subtitle**: Topic 9: Databases | The Architect

## 2. Hook
- **Slide Type**: Hook
- **Question**: How does a database start?
- **Content**:
    - Before we can query data, we must **build** the structure.
    - This is called **DDL** (Data Definition Language).
    - It's like drawing the blueprints before building the house.

## 3. Creating Structures
- **Slide Type**: Concept (Vertical Stack)
- **Question**: How do we make a table?
- **Content**:
    - **CREATE DATABASE**: Makes the container.
    - **CREATE TABLE**: Defines the columns and types.
    - **Syntax**:
      ```sql
      CREATE TABLE Student (
        StudentID INT,
        Name VARCHAR(50),
        DOB DATE
      );
      ```

### Vertical: Data Types in SQL
- **Slide Type**: Deep Dive
- **Content**:
    - **CHAR(n)**: Fixed length (e.g., Postcode).
    - **VARCHAR(n)**: Variable length (e.g., Name).
    - **INT**: Whole numbers.
    - **DATE**: Dates (YYYY-MM-DD).
    - **BOOLEAN**: True/False.

## 4. Modifying Structures
- **Slide Type**: Concept
- **Question**: What if we forgot something?
- **Content**:
    - **ALTER TABLE**: Changes the structure.
    - **ADD**: Adds a new column.
    - **ADD PRIMARY KEY**: Sets the unique identifier.
    - **Example**:
      ```sql
      ALTER TABLE Student
      ADD PRIMARY KEY (StudentID);
      ```

## 5. Adding Data
- **Slide Type**: Concept
- **Question**: How do we put data in?
- **Content**:
    - **INSERT INTO**: Adds a new record.
    - **Syntax**:
      ```sql
      INSERT INTO Student (StudentID, Name)
      VALUES (1, 'John Smith');
      ```

## 6. Plenary
- **Slide Type**: Plenary (True/False)
- **Content**:
    1. "VARCHAR is for fixed length text." (False - Variable).
    2. "ALTER TABLE is used to add data." (False - Change structure. INSERT adds data).
    3. "You must define data types when creating a table." (True).
