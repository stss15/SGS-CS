Topic 9: Databases

Complete Specification & Learning Guide

Cambridge IGCSE Computer Science 0478 (2026–2028)

Organising the World's Information

From social media profiles to school attendance records, databases are the invisible engines that store and retrieve the world's data. In this topic, you will learn how to structure data efficiently using tables, enforce accuracy with data types and validation, and retrieve exactly what you need using the powerful language of SQL.

What You Will Learn

This topic focuses on the design and querying of single-table databases:

  • Database Structure: Understanding Fields (columns), Records (rows), and the concept of a Flat-File database.
  • Data Types: Choosing the correct type for data (Text, Integer, Real, Boolean, Date/Time).
  • Primary Keys: The crucial role of a unique identifier to distinguish every record.
  • SQL (Structured Query Language): Writing scripts to Select, Filter, Order, and Count data.

9.1 Database Concepts

What you need to know:

  • Definition: Define a single-table database from given data storage requirements.
  • Structure: Understand the meaning of:
    • Field: A single category of data (a column).
    • Record: A complete set of data for one item (a row).
    • Validation: Rules to ensure data entered is reasonable.
  • Data Types: Suggest suitable basic data types:
    • Text/Alphanumeric
    • Character
    • Boolean
    • Integer
    • Real
    • Date/Time
  • Primary Key: Understand the purpose of a primary key and identify a suitable primary key for a given database table.

Notes & Guidance

  • Single Table: For IGCSE, we focus on "Flat File" databases (just one table), not complex relational databases with foreign keys.
  • Primary Key: Must be unique for every record. Examples: StudentID, VehicleReg, ISBN. Never use names (people can have the same name).
  • Data Types:
    • Text: "John", "CV34 5HT" (Phone numbers are text!).
    • Real: 12.99, 3.14 (Numbers with decimals).
    • Boolean: True/False, Yes/No.

9.2 Structured Query Language (SQL)

What you need to know:

  • Querying: Read, understand, and complete SQL scripts to query data stored in a single database table.
  • Commands: You must be able to use and understand the following commands:
    • SELECT (specifies the fields to show)
    • FROM (specifies the table)
    • WHERE (specifies the criteria/filter)
    • ORDER BY (sorts the results)
    • DESCENDING / ASCENDING
    • SUM (adds up values)
    • COUNT (counts number of records)
  • Operators: Use logical operators within the WHERE clause:
    • AND
    • OR

Notes & Guidance

  • Standard Query Structure:
    SELECT Name, Age
    FROM Students
    WHERE Age >= 15 AND House = 'Red'
    ORDER BY Age DESCENDING
  • Wildcard: SELECT * means "Select ALL fields".
  • Functions: SELECT COUNT(ID) FROM Students returns a single number (the total count).
  • Ordering: The default is usually Ascending, but it is good practice to specify it explicitly if required.

Exam Expectations: What You Need to Do

For the Cambridge IGCSE (0478) exam, you must be able to:

  • Write SQL: Write a full SQL query from scratch based on a scenario (e.g., "Show the names of all cars that are Blue and cost more than £10,000").
  • Interpret SQL: Look at a given SQL script and predict exactly what the output will be (which records will be shown and in what order).
  • Choose Data Types: Given a sample table of data (e.g., a flight schedule), identify the most appropriate data type for each column (e.g., Arrival Time = Date/Time, Flight Number = Text).
  • Identify Keys: Select the best field to use as a Primary Key and explain why (uniqueness).
