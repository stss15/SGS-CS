# 9.1 Databases Slide Plan

## 1. Title Slide
- **Slide Type**: Title
- **Title**: 9.1 Databases
- **Subtitle**: Topic 9: Databases | Organising the World's Information

## 2. Hook
- **Slide Type**: Hook
- **Question**: How does Instagram find your profile in milliseconds?
- **Content**:
    - It's not magic; it's a **Database**.
    - A structured collection of data that allows for efficient storage and retrieval.
    - Without databases, the internet would be a chaotic mess of files.
- **Visuals**: Icon of a messy pile of papers vs a neat filing cabinet.

## 3. Database Structure
- **Slide Type**: Concept (Vertical Stack)
- **Question**: What is a database actually made of?
- **Content**:
    - **Single-Table Database**: For IGCSE, we look at "Flat File" databases (one table).
    - **Table**: Contains data about one type of item (e.g., Patients, Cars).
    - **Record (Row)**: A complete set of data for one item.
    - **Field (Column)**: A single category of data.
- **Visuals**: Diagram of a table with "Field" pointing to a column and "Record" pointing to a row.

### Vertical: Anatomy of a Table
- **Slide Type**: Deep Dive
- **Content**:
    - **Field Names**: Must be meaningful (e.g., `FirstName`, `DateOfAdmission`).
    - **Validation**: Rules to ensure data is reasonable (e.g., Ward Number 1-10).
    - **Consistency**: Every record has the same fields.

## 4. Data Types
- **Slide Type**: Concept (Vertical Stack)
- **Question**: Why can't we just store everything as text?
- **Content**:
    - Computers need to know *how* to process the data.
    - You can't multiply "Dave" by 5.
    - **Data Type**: Classifies how data is stored, displayed, and processed.

### Vertical: The 6 Key Types
- **Slide Type**: Deep Dive
- **Content**:
    - **Text/Alphanumeric**: "John", "CV34 5HT" (Phone numbers are text!).
    - **Character**: Single letter/symbol ('M', 'F').
    - **Integer**: Whole number (15, -20).
    - **Real**: Decimal number (12.99, 3.14).
    - **Boolean**: True/False, Yes/No.
    - **Date/Time**: 22/11/2022, 14:30.

## 5. Primary Keys
- **Slide Type**: Concept
- **Question**: How do we tell two "John Smiths" apart?
- **Content**:
    - **The Problem**: Names are not unique.
    - **The Solution**: A **Primary Key**.
    - **Definition**: A field that uniquely identifies every record in a table.
    - **Examples**: `StudentID`, `VehicleReg`, `ISBN`, `PassportNumber`.
- **Visuals**: Two people named "John Smith" with different ID cards.

## 6. Plenary
- **Slide Type**: Plenary (True/False)
- **Content**:
    1. "A phone number should be stored as an Integer." (False - it starts with 0 and isn't used for maths. Use Text).
    2. "A Primary Key can be the same for two records." (False - must be unique).
    3. "A Boolean data type stores text." (False - True/False).
    4. "A single-table database is also called a Flat File." (True).
