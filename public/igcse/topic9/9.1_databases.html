What is a database?

What is a Database?

Definition: A database is a persistent, structured collection of data that allows people to extract information.

Persistent

Data is saved permanently (not lost when turned off)

Structured

Organised in a specific way (rows and columns)

Searchable

Easy to find and filter data

Database Structure

What's inside a database?

Analogy: Think of a spreadsheet!

A database is made up of tables with rows and columns.

See the parts

The Three Key Parts

Table

A collection of related data about one thing.

e.g., Students, Books, Orders

Record (Row)

One complete entry in the table.

e.g., One specific student

Field (Column)

A single category of information.

e.g., Name, Age, ID

Example Table: Students

StudentID FirstName Surname Age House
S001 Ali Khan 14 Red
S002 Sara Jones 15 Blue
S003 John Smith 14 Red

Fields = Columns (StudentID, FirstName, Surname, Age, House)

Records = Rows (each student is one record)

Data Types

Why can't we store everything as text?

The database needs to know what kind of data to expect.

You can't sort "twenty" before "five" as numbers!

The main types

Common Data Types

Type Description Examples
Text / String Letters, numbers, symbols "John", "AB12 3CD"
Integer Whole numbers 15, -3, 0
Real Decimal numbers 3.14, 99.99
Boolean True or False only TRUE, FALSE
Date/Time Dates and times 15/03/2024, 14:30

Common Trap!

What data type should a phone number be?

Wrong: Integer

07123456789 would lose the leading 0!

We don't do maths with phone numbers.

Correct: Text

Keeps the leading 0.

Phone numbers aren't used for calculations.

Primary Keys

How do we tell two "John Smiths" apart?

Names are not unique. Two people can have the same name, birthday, and even address!

The solution

The Primary Key

Definition: A primary key is a field that uniquely identifies every record in a table.

Rules

  • Must be unique
  • Cannot be empty
  • Should never change

Examples

  • StudentID (S001, S002...)
  • ISBN (for books)
  • OrderNumber
  • NI Number

Single-Table Database

For IGCSE, we focus on flat file (single-table) databases.

Flat File

All data in one table.

Simple but can have repeated data.

What we use at IGCSE.

Relational

Data split across multiple linked tables.

More complex but avoids repetition.

A-Level topic!

Key Takeaways

  1. A database is a structured collection of data
  2. Tables contain records (rows) and fields (columns)
  3. Each field has a data type (Text, Integer, Boolean, etc.)
  4. A primary key uniquely identifies each record
  5. Phone numbers should be stored as Text, not Integer!

Question 1

A phone number should be stored as an Integer.

Answer: FALSE

Phone numbers should be Text. Integers would lose leading zeros, and we don't do maths with phone numbers!

Question 2

A primary key can have the same value for two records.

Answer: FALSE

A primary key must be unique for every record.

Question 3

A record is a row in a table.

Answer: TRUE

A record (row) contains all the data for one item. A field is a column.
