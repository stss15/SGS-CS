The Big Question

What's the difference between a wrong answer and a silly answer?

Question: What is your age?

Wrong Answer

"I'm 25" (but really 26)

It's plausible, just incorrect

Silly Answer

"I'm 500 years old"

Obviously not reasonable!

How do we catch these?

Two Types of Checking

Validation

Is it reasonable?

Automated checks done by the computer.

Catches "silly" data.

Verification

Is it correct?

Checking by a human or double-entry.

Catches "wrong" data.

Key: Validation does NOT mean data is correct – just that it's reasonable!

Validation Checks

What automatic checks can a program do?

Range Check

Length Check

Type Check

Presence Check

Format Check

Check Digit

Learn each one

Range Check

Definition: Ensures data is within a specified range of values.

Example

Age must be between 0 and 120

Exam mark must be 0-100

Pseudocode

IF Age >= 0 AND Age <= 120 THEN
    Accept data
ELSE
    Reject data
ENDIF

Length Check

Definition: Ensures data has the correct number of characters.

Example

Password must be at least 8 characters

Phone number must be 11 digits

Pseudocode

IF LENGTH(Password) >= 8 THEN
    Accept data
ELSE
    Reject data
ENDIF

Type Check

Definition: Ensures data is the correct data type.

Example

Age must be an integer (not "twenty")

Name must be text (not 12345)

Real Example

Price field:

29.99

"twenty nine"

Presence Check

Definition: Ensures the field is not left empty.

Example

Email address is required

Name cannot be blank

Pseudocode

IF Name != "" THEN
    Accept data
ELSE
    OUTPUT "Name required"
ENDIF

Format Check

Definition: Ensures data matches a specific pattern.

Example

Email must contain @ and .

Date must be DD/MM/YYYY

Postcode: AA00 0AA

Valid vs Invalid

user@email.com

useremail

Check Digit

Definition: An extra digit calculated from the other digits to detect errors.

Where are they used?

Barcodes

ISBN (books)

Card numbers

How it works: A calculation is done on all the digits. If someone enters a number incorrectly, the check digit won't match.

Verification

How do we check data is actually correct?

Validation can't tell if "John" should be "Jon".

Both are reasonable – but one might be wrong!

Two methods

Verification Methods

Double Entry

Enter the data twice.

The computer compares both entries.

Used for: Passwords, emails

Visual Check

Display data back to the user.

Ask them to confirm it's correct.

Used for: Delivery addresses, orders

Double Entry Example

OUTPUT "Enter your email:"
INPUT Email1
OUTPUT "Confirm your email:"
INPUT Email2
IF Email1 = Email2 THEN
    OUTPUT "Email confirmed"
ELSE
    OUTPUT "Emails do not match!"
ENDIF

Why this works: It's unlikely you'd make the same typo twice!

Validation vs Verification

Validation Verification
Question Is it reasonable? Is it correct?
Done by Computer (automatic) Human or double-entry
Example Age can't be 500 Is it really 25, not 26?
Can catch Silly mistakes Typing errors

Key Takeaways

  1. Validation = Is it reasonable? (automatic)
  2. Verification = Is it correct? (human/double-entry)
  3. Validation checks: Range, Length, Type, Presence, Format, Check Digit
  4. Verification methods: Double Entry, Visual Check

True or False?

Check Your Understanding

  1. Validation guarantees that data is correct.
  2. A range check ensures data is within specific limits.
  3. Double entry is a type of verification.

Press Down to reveal answers

Question 1

Validation guarantees that data is correct.

Answer: FALSE

Validation only checks if data is reasonable, not if it's actually correct. An age of 25 passes validation even if you're really 26.

Question 2

A range check ensures data is within specific limits.

Answer: TRUE

A range check verifies that data falls within a defined minimum and maximum value.

Question 3

Double entry is a type of verification.

Answer: TRUE

Double entry is a verification method that checks if data was entered correctly by comparing two inputs.
