Flowcharts

A visual way to show an algorithm

A flowchart uses shapes connected by arrows to show the sequence of steps.

Key Benefit: You can "see" the logic at a glance!

Pseudocode

A text-based way to write algorithms

Pseudocode = "fake code"

It looks like a programming language but isn't tied to any specific one.

Key Benefit: Easy to read and write, structured like real code!

Learn the keywords

Essential Pseudocode Keywords

Input/Output

INPUT Name
OUTPUT "Hello"

Assignment

Total ← 10
Name ← "Ali"

Use ← (arrow) to assign values

Selection (IF)

IF condition THEN
    ...
ELSE
    ...
ENDIF

Iteration (FOR)

FOR i ← 1 TO 10
    ...
NEXT i

Pseudocode Example

The same "add two numbers" algorithm

INPUT Num1
INPUT Num2
Total ← Num1 + Num2
OUTPUT Total

Notice: No strict syntax rules – just clear, structured instructions!

With Selection (IF)

Problem: Check if a number is positive or negative

INPUT Number
IF Number >= 0 THEN
    OUTPUT "Positive"
ELSE
    OUTPUT "Negative"
ENDIF

Key: Always close IF with ENDIF

With Iteration (Loop)

Problem: Output numbers 1 to 5

FOR Count ← 1 TO 5
    OUTPUT Count
NEXT Count

Output: 1, 2, 3, 4, 5

The loop repeats 5 times, changing Count each time.

Flowchart vs Pseudocode

Flowcharts

  • Visual – easy to see flow
  • Good for complex logic
  • Time-consuming to draw
  • Hard to change later

Pseudocode

  • Quick to write
  • Easy to modify
  • Closer to real code
  • Less visual

In exams: You must know both!

Key Takeaways

  1. Flowcharts use shapes and arrows to show logic visually
  2. Pseudocode uses structured text like a programming language
  3. Key symbols: Terminator, Process, I/O, Decision
  4. Key keywords: INPUT, OUTPUT, IF/THEN/ELSE, FOR/NEXT

Press Down to reveal answers

Question 1

A diamond shape represents a decision in a flowchart.

Answer: TRUE

The diamond shape is used for Yes/No questions (decisions/IF statements).

Question 2

Pseudocode must follow strict syntax rules to work.

Answer: FALSE

Pseudocode is flexible – it's not a real programming language, so there are no strict syntax rules.

Question 3

The arrow symbol (←) is used for assignment in pseudocode.

Answer: TRUE

We use ← to show that a value is being stored in a variable. e.g., Total ← 10
