The Big Question

How do you find a bug when the code "looks right"?

Sometimes code seems perfect, but gives wrong answers.

You need to think like a computer – execute it step by step.

This process is called a dry run, and we record it in a trace table.

What is a trace table?

What is a Trace Table?

Definition: A table used to record the values of variables as an algorithm is executed, step by step.

Structure

One column per variable

Plus an OUTPUT column

One row per change

Purpose

Track how values change

Find logic errors

Understand algorithm flow

Let's Trace This Code

B ← 3
C ← A + B
OUTPUT C

Tracing Loops

This is where trace tables really shine!

Total ← 0
FOR Count ← 1 TO 3
    Total ← Total + Count
NEXT Count
OUTPUT Total
Watch each iteration

Tracing the Loop

Total ← 0
FOR Count ← 1 TO 3
    Total ← Total + Count
NEXT Count
OUTPUT Total
Count Total OUTPUT
0
1 1
2 3
3 6
3 6 6

Logic: Total = 0 + 1 + 2 + 3 = 6

Trace Table Golden Rules

  1. One column per variable – plus OUTPUT
  2. New row only when a value changes
  3. Execute in order – line by line
  4. Show your working – especially in loops
  5. Don't skip steps – even if it seems obvious

A Bigger Example

Max ← 0
FOR i ← 1 TO 4
    INPUT Num
    IF Num > Max THEN
        Max ← Num
    ENDIF
NEXT i
OUTPUT Max

Key Takeaways

  1. A trace table tracks variable values as code runs
  2. A dry run = executing code manually on paper
  3. One column per variable, plus OUTPUT
  4. New row each time a value changes
  5. Essential for finding logic errors in loops!

True or False?

Check Your Understanding

Press Down to reveal answers

Question 1

A trace table should have one column for each variable.

Answer: TRUE

Each variable gets its own column, plus an OUTPUT column to record any outputs.

Question 2

You add a new row every line of code.

Answer: FALSE

You add a new row only when a value changes, not for every line of code.

Question 3

Trace tables are only useful for finding syntax errors.

Answer: FALSE

Trace tables are used to find logic errors – when code runs but produces wrong results. Syntax errors stop code from running at all!
