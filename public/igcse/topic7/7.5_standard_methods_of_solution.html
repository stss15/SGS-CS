Totalling

How do we add up a list of numbers?

Analogy: Adding items to a shopping cart.

Start with £0, then add each item's price as you go.

See the pattern

Totalling Pattern

Total ← 0
FOR Count ← 1 TO 5
    INPUT Number
    Total ← Total + Number
NEXT Count
OUTPUT Total

Key Step 1

Total ← 0

Initialize - start with an empty total

Key Step 2

Total ← Total + Number

Accumulate - add each value

Counting

How do we count items that match a condition?

Example: Count how many students passed (mark ≥ 50)

See the pattern

Counting Pattern

PassCount ← 0
FOR Student ← 1 TO 30
    INPUT Mark
    IF Mark >= 50 THEN
        PassCount ← PassCount + 1
    ENDIF
NEXT Student
OUTPUT PassCount

Key Step 1

PassCount ← 0

Initialize - start at zero

Key Step 2

PassCount ← PassCount + 1

Increment - add 1 each time

Finding Maximum & Minimum

How do we find the highest (or lowest) value?

Analogy: Finding the tallest person in a room.

Assume the first person is tallest. Then compare each new person – if taller, they become the new record holder.

See the pattern

Maximum Pattern

INPUT FirstNumber
Max ← FirstNumber
FOR Count ← 2 TO 10
    INPUT Number
    IF Number > Max THEN
        Max ← Number
    ENDIF
NEXT Count
OUTPUT "Highest: ", Max

Key: Set Max to the first value, then compare each new value. If bigger, update Max.

Minimum Pattern

INPUT FirstNumber
Min ← FirstNumber
FOR Count ← 2 TO 10
    INPUT Number
    IF Number < Max THEN
        Min ← Number
    ENDIF
NEXT Count
OUTPUT "Lowest: ", Min

Same pattern! Just change > to <

Finding the Average

How do we calculate an average?

Average = Total ÷ Count

First total all values, then divide by how many there are.

Total ← 0
FOR Count ← 1 TO 10
    INPUT Number
    Total ← Total + Number
NEXT Count
Average ← Total / 10
OUTPUT Average

Linear Search

How do we find a specific item in a list?

Analogy: Finding a book on a messy bookshelf.

Start at the beginning, check each book one by one until you find it.

See the pattern

Linear Search Pattern

INPUT SearchValue
Found ← FALSE
Position ← 0
FOR Index ← 1 TO 10
    IF List[Index] = SearchValue THEN
        Found ← TRUE
        Position ← Index
    ENDIF
NEXT Index
IF Found = TRUE THEN
    OUTPUT "Found at position ", Position
ELSE
    OUTPUT "Not found"
ENDIF

Key: Check each item in turn. Use a Found flag to remember if we found it.

Bubble Sort

How do we put a list in order?

Analogy: Bubbles rising to the top of a glass.

Compare pairs of items. If they're in the wrong order, swap them. Keep going until no more swaps are needed.

Watch it work

Bubble Sort Demo

Sort this list: [5, 3, 8, 1]

Pass 1:

[5, 3, 8, 1] → [3, 5, 8, 1] (swap)

[3, 5, 8, 1] → [3, 5, 8, 1] (no swap)

[3, 5, 8, 1] → [3, 5, 1, 8] (swap)

Pass 2:

[3, 5, 1, 8] → [3, 5, 1, 8] (no swap)

[3, 5, 1, 8] → [3, 1, 5, 8] (swap)

Pass 3:

[3, 1, 5, 8] → [1, 3, 5, 8] (swap)

Result: [1, 3, 5, 8] ✓ Sorted!

Bubble Sort Pseudocode

REPEAT
    Swapped ← FALSE
    FOR Index ← 1 TO Length - 1
        IF List[Index] > List[Index + 1] THEN
            Temp ← List[Index]
            List[Index] ← List[Index + 1]
            List[Index + 1] ← Temp
            Swapped ← TRUE
        ENDIF
    NEXT Index
UNTIL Swapped = FALSE

Key: Keep making passes until no swaps happen (Swapped = FALSE).

Key Takeaways

  1. Totalling: Initialize to 0, then add each value
  2. Counting: Initialize to 0, then add 1 for each match
  3. Max/Min: Set to first value, compare and update
  4. Average: Total ÷ Count
  5. Linear Search: Check each item one by one
  6. Bubble Sort: Compare pairs and swap until sorted

True or False?

Check Your Understanding

  1. When totalling, you should initialize the total to 1.
  2. Linear search checks every item until it finds a match.
  3. Bubble sort stops when no swaps are made in a pass.

Press Down to reveal answers

Question 1

When totalling, you should initialize the total to 1.

Answer: FALSE

Always initialize to 0. If you started at 1, your total would be 1 more than it should be!

Question 2

Linear search checks every item until it finds a match.

Answer: TRUE

Linear search goes through items one by one from the start until it finds what it's looking for (or reaches the end).

Question 3

Bubble sort stops when no swaps are made in a pass.

Answer: TRUE

If no swaps are made, the list must already be sorted, so we can stop!
