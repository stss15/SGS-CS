The Big Question

How would you teach a robot to make tea?

A robot doesn't know what "make tea" means. You need to give exact, step-by-step instructions.

Those step-by-step instructions are called an algorithm.

Let's define it properly

What is an Algorithm?

Definition: An algorithm is an ordered set of steps to solve a problem.

Ordered

Steps must be in a specific sequence

Precise

No ambiguity – each step is clear

Finite

It must eventually end

Example: The Tea Algorithm

Step-by-step instructions for making tea

Watch each step

Building the Algorithm

1. Fill kettle with water

2. Turn kettle on

3. Wait until water boils

4. Put teabag in cup

5. Pour water into cup

6. Wait 2 minutes

7. Remove teabag

8. Add milk (if wanted)

Why Order Matters

Correct Order

1. Put teabag in cup

2. Pour hot water

3. Remove teabag

Wrong Order

1. Remove teabag

2. Pour hot water

3. Put teabag in cup

Key Point: Algorithms must be in the correct sequence to work!

Decomposition in Algorithms

What if one step is too complex?

Example: "Fill kettle with water" is actually multiple steps:

  1. Pick up kettle
  2. Take to tap
  3. Open lid
  4. Turn on tap
  5. Wait until full
  6. Turn off tap
  7. Close lid
This is Decomposition

When to Decompose

Rule of Thumb: If a step could be misunderstood or needs sub-steps, break it down.

Simple Enough

"Turn on kettle"

Clear, single action

Needs Decomposition

"Make breakfast"

Too vague, many sub-tasks

Explaining Algorithms

Can you describe what this does in plain English?

Total ← 0
INPUT Number1
INPUT Number2
Total ← Number1 + Number2
OUTPUT Total
Let's break it down

Step-by-Step Explanation

Line What it does
Total ← 0 Create a storage space called Total, start at 0
INPUT Number1 Ask the user to enter the first number
INPUT Number2 Ask the user to enter the second number
Total ← Number1 + Number2 Add both numbers and store in Total
OUTPUT Total Display the result to the user

Purpose: This algorithm adds two numbers and shows the result.

A Bigger Example

What does this algorithm do?

Total ← 0
FOR Count ← 1 TO 5
    INPUT Mark
    Total ← Total + Mark
NEXT Count
Average ← Total / 5
OUTPUT Average
Break it down

Explanation

Setup

Total ← 0

Start with an empty total

Loop

FOR Count ← 1 TO 5

Repeat 5 times

Inside Loop

INPUT Mark

Total ← Total + Mark

Get a mark and add it to Total

After Loop

Average ← Total / 5

OUTPUT Average

Calculate and show average

Purpose: Input 5 marks, calculate and display the average.

Identifying Inputs, Processes, Outputs

Total ← 0
FOR Count ← 1 TO 5
    INPUT Mark
    Total ← Total + Mark
NEXT Count
Average ← Total / 5
OUTPUT Average

Inputs

Mark (entered 5 times)

Processes

Add marks together

Divide by 5

Outputs

Average

Your Turn

Explain what this algorithm does:

INPUT Temperature
IF Temperature > 30 THEN
    OUTPUT "It's hot!"
ELSE
    OUTPUT "It's not hot"
ENDIF
Check your answer

Answer

  1. INPUT Temperature – Ask user to enter a temperature
  2. IF Temperature > 30 – Check if it's above 30
  3. THEN OUTPUT "It's hot!" – If yes, display "It's hot!"
  4. ELSE OUTPUT "It's not hot" – If no, display "It's not hot"

Purpose: Get a temperature and tell the user if it's hot (above 30°) or not.

Input

Temperature

Process

Compare to 30

Output

A message

Key Takeaways

  1. An algorithm is an ordered set of steps to solve a problem
  2. Algorithms must be precise, ordered, and finite
  3. Use decomposition to break complex steps into simpler ones
  4. Always identify the Inputs, Processes, and Outputs

Question 1

The order of steps in an algorithm doesn't matter.

Answer: FALSE

Algorithms must be ordered – the sequence of steps is crucial!

Question 2

An algorithm must eventually stop.

Answer: TRUE

Algorithms must be finite – they must reach an end.

Question 3

Decomposition means making steps more complex.

Answer: FALSE

Decomposition means breaking down complex steps into simpler, more manageable ones.
