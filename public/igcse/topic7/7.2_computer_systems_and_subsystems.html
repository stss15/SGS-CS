This is Top-Down Design

Top-Down Design

Definition: A method where we break a large system into smaller sub-systems, and keep breaking down until each part is simple enough to understand.

Key Idea: Work from the top (the whole problem) down to the small details.

But how do we show this breakdown visually?

Structure Diagrams

How do we visualise a system and its sub-systems?

A Structure Diagram is a hierarchical chart showing:

  • The main system at the top
  • Each sub-system branching below
  • More detail as you go down
Let's build one step-by-step

Storage = data that's kept between runs of the program (saved to files, databases, etc.)

Real Example: ATM Machine

Let's analyse an ATM using Inputs, Processes, Outputs, Storage

Break it down

ATM: Inputs

What data goes INTO the ATM?

Card inserted

PIN entered

Amount requested

ATM: Processes

What does the ATM DO with that data?

  • Validate the PIN against stored data
  • Check if sufficient balance exists
  • Calculate new balance after withdrawal
  • Count the cash to dispense

ATM: Outputs & Storage

Outputs

  • Cash dispensed
  • Receipt printed
  • Messages on screen
  • Card returned

Storage

  • Account balances
  • PIN numbers
  • Transaction history

Key Takeaways

  1. Top-Down Design = Break big problems into sub-systems
  2. Structure Diagrams = Visualise the hierarchy
  3. Every sub-system has: Inputs, Processes, Outputs, Storage
  4. Keep breaking down until each part is simple enough to code
