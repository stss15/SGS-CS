Thinking Like a Computer

Before a single line of code is written, a problem must be understood and solved logically. This topic covers the art of Computational Thinking: breaking big problems down into small pieces (decomposition), designing step-by-step solutions (algorithms), and ensuring those solutions are robust (testing and validation). This is the foundation of all programming.

What You Will Learn

This topic prepares you for the practical aspects of Paper 2:

  • The Development Cycle: The four key stages: Analysis, Design, Coding, and Testing.
  • Decomposition: How to break computer systems into sub-systems and component parts.
  • Standard Algorithms: Memorising the logic for Linear Search, Bubble Sort, Totalling, Counting, and finding Averages/Max/Min.
  • Validation & Verification: Ensuring data input is both reasonable (Validation) and consistent (Verification).
  • Testing: Using Trace Tables and different types of test data (Normal, Abnormal, Extreme, Boundary) to prove your algorithm works.

7.1 Program Development Life Cycle

What you need to know:

  • The Cycle: Understand the program development life cycle, limited to: Analysis, Design, Coding, and Testing.
  • Decomposition:
    • (a) Understand that every computer system is made up of sub-systems, which are made up of further sub-systems.
    • (b) Understand how a problem can be decomposed into its component parts (inputs, processes, outputs, storage).
  • Design Methods: Use different methods to design and construct a solution to a problem (Structure diagrams, Flowcharts, Pseudocode).

Notes & Guidance

  • Analysis: Identifying the problem and requirements. Involves abstraction and decomposition.
  • Design: Planning the solution using structure diagrams, flowcharts, and pseudocode.
  • Coding: Writing the actual program code and performing iterative testing.
  • Testing: Testing the program code with the use of specific test data.
  • Structure Diagrams: Hierarchical diagrams showing how a system is broken down into subsystems.

7.2 Standard Methods of Solution

What you need to know:

  • Standard Algorithms: Understand and be able to write standard methods of solution limited to:
    • Linear Search
    • Bubble Sort
    • Totalling
    • Counting
    • Finding Maximum, Minimum, and Average values
  • Purpose: Explain the purpose of a given algorithm and describe the processes involved in it.

Notes & Guidance

  • Linear Search: Checking each item in a list one by one until the item is found or the end of the list is reached.
  • Bubble Sort: Comparing adjacent items and swapping them if they are in the wrong order. This pass is repeated until no swaps are needed.
  • Totalling: Using a variable (e.g., Total) to keep a running sum inside a loop (Total = Total + Number).
  • Counting: Using a variable (e.g., Count) to count occurrences (Count = Count + 1).

7.3 Validation, Verification and Testing

What you need to know:

  • Validation: Understand the need for validation checks on input data and the different types:
    • Range check, Length check, Type check, Presence check, Format check, Check digit.
  • Verification: Understand the need for verification checks and the different types:
    • Visual check, Double entry check.
  • Test Data: Suggest and apply suitable test data:
    • Normal, Abnormal, Extreme, Boundary.
  • Trace Tables: Complete a trace table to document a dry-run of an algorithm.
  • Errors: Identify errors in given algorithms and suggest ways of correcting them. Write and amend algorithms using pseudocode, program code, and flowcharts.

Notes & Guidance

  • Validation vs Verification: Validation checks if data is reasonable/sensible. Verification checks if data was copied correctly from the source.
  • Test Data Types:
    • Normal: Data that should be accepted (e.g., 50 for a percentage).
    • Abnormal: Data that should be rejected (e.g., "ABC" or 150 for a percentage).
    • Extreme: The largest/smallest acceptable values (e.g., 0 and 100).
    • Boundary: The extreme values AND the values immediately outside the limit (e.g., -1, 0, 100, 101).
  • Dry Runs: You must be able to manually step through an algorithm and record the value of variables in a trace table at each step.

Exam Expectations: What You Need to Do

For the Cambridge IGCSE (0478) exam, you must be able to:

  • Draw Flowcharts: Create accurate flowcharts using the correct symbols (terminator, process, input/output, decision, flow line).
  • Write Pseudocode: Write clear pseudocode for a given problem (e.g., "Write an algorithm to input 10 numbers and output the average").
  • Trace Logic: Complete a trace table for a complex loop to show you understand exactly how the variables change.
  • Spot Errors: Look at a broken piece of code (e.g., an infinite loop or incorrect logic) and identify exactly what is wrong and how to fix it.
  • Define Checks: Clearly distinguish between a range check (is it between 1-100?) and a length check (is it 8 characters long?).
