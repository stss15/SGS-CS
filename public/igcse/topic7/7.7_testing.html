Choosing the Right Test Data

Quick Recall (7.6)

Q1: What is validation?

Automatic checks that data is reasonable

Q2: Name 2 validation checks.

Range, Length, Type, Presence, Format, Check Digit

The Big Question

Why do car companies crash their own cars?

They're not trying to destroy them – they're testing them!

They test with normal driving, extreme speeds, and edge cases.

Same with software: We need to test with different types of data to make sure our program works in ALL situations.

Four types of test data

Types of Test Data

Normal Data

Typical, expected data

Abnormal Data

Data that should be rejected

Extreme Data

At the very edges of valid

Boundary Data

Right at the boundary

Let's Use an Example

A program accepts exam marks from 0 to 100

INPUT Mark
IF Mark >= 0 AND Mark <= 100 THEN
    OUTPUT "Valid mark"
ELSE
    OUTPUT "Invalid mark"
ENDIF

What test data should we use to test this properly?

Normal Data

Typical data that should be accepted

Definition

Normal data is typical, sensible data that the program should accept and process correctly.

Examples (0-100)

45

73

50

All within the valid range

Abnormal Data

Data that should be rejected

Definition

Abnormal data (also called erroneous data) is data the program should NOT accept.

The program should reject it without crashing!

Examples (0-100)

-5 (below range)

150 (above range)

"fifty" (wrong type)

Extreme Data

Data at the edges of valid range (but still valid)

Definition

Extreme data is at the very limits of what's accepted.

Tests the edges of your validation.

Examples (0-100)

0 (minimum valid)

100 (maximum valid)

Both should be accepted!

Boundary Data

Data right at the boundary between valid and invalid

Definition

Boundary data tests the exact point where data changes from valid to invalid.

Always test in pairs: one that passes, one that fails.

Examples (0-100)

Lower boundary:

-1 (just below)

0 (just passes)

Upper boundary:

100 (just passes)

101 (just above)

Key Takeaways

  1. Normal = Typical data that should work
  2. Abnormal = Data that should be rejected
  3. Extreme = At the very edges of valid (but still valid)
  4. Boundary = Pairs at the exact boundary (one valid, one invalid)
  5. Always test with all four types!

Question 1

Extreme data should be rejected by the program.

Answer: FALSE

Extreme data is at the edges but still valid. It should be accepted!

Question 2

Boundary data tests values at the very edge of valid and invalid.

Answer: TRUE

Boundary data tests the exact point where data goes from valid to invalid (both sides of the boundary).

Question 3

You only need to test with normal data.

Answer: FALSE

You need to test with all four types: normal, abnormal, extreme, and boundary data to test thoroughly.
