# Slide Plan: 7.9 Writing and Amending Algorithms

**Goal:** Apply the full design-build-test cycle to write clear pseudocode/flowcharts from a brief, amend them when requirements change, and check with test data.

**Pedagogy:** Architect mindset. Follow an 8-step scaffold (spec → design → pseudocode → test) using a single running brief. Then practise amendments to show agility.

## 3. The 8-Step Build Process
- **Type:** Concept (Horizontal)
- **Content:**
  - **Steps:**
    1. Specify the problem (inputs, outputs, rules, constraints).
    2. Decompose into sub-problems.
    3. Decide data (types, lists start at index 1).
    4. Draw a structure diagram.
    5. Sketch flowchart or pseudocode.
    6. Refine for readability (meaningful names, indent).
    7. Test with Normal/Abnormal/Boundary data (trace table).
    8. Fix errors and iterate.

### 3.1 Vertical Deep Dive: Worked Example (Ticket Sales)
- **Type:** Vertical Stack
- **Slide 1 (Brief):**
  - Tickets £20 each; 10+ → 10% off; 20+ → 20% off; max 25 tickets.
- **Slide 2 (Design):**
  - IPO: Input `NumberOfTickets`; Process: validate 1–25, choose discount, calculate cost; Output: cost.
  - Structure diagram: Validate → Calculate Discount → Calculate Cost → Output.
- **Slide 3 (Pseudocode - 1-based friendly):**
    ```
    REPEAT
      INPUT NumberOfTickets
    UNTIL NumberOfTickets > 0 AND NumberOfTickets < 26
    IF NumberOfTickets < 10 THEN
      Discount <- 0
    ELSEIF NumberOfTickets < 20 THEN
      Discount <- 0.1
    ELSE
      Discount <- 0.2
    ENDIF
    Cost <- NumberOfTickets * 20 * (1 - Discount)
    OUTPUT "Your tickets cost ", Cost
    ```
- **Slide 4 (Test Data Table):**
  - Show Normal (1, 15), Boundary (0, 10, 20, 25, 26), Abnormal (“five”). Record expected outputs.
