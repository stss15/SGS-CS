Topic 1: Data Representation

Complete Specification & Learning Guide

Cambridge IGCSE Computer Science 0478 (2026–2028)

Welcome to the Language of Computers

Everything a computer processes—from your favourite song to the code running this website—boils down to ones and zeros. In this unit, we peel back the layers to understand how computers translate our complex world (numbers, text, sound, and images) into simple binary states.

What You Will Learn

In this section, we cover the fundamental building blocks of computer science:

  • Number Systems: How to navigate Binary (Base-2), Denary (Base-10), and Hexadecimal (Base-16).
  • Binary Arithmetic: The rules for adding binary numbers, performing logical shifts, and Two's Complement.
  • Multimedia Representation: Text, Sound, and Images.
  • Data Storage: Measurement units and compression.

1.1 Number Systems

What you need to know:

  • Binary Usage: Understand how and why computers use binary to represent all forms of data.
  • Number Systems:
    • (a) Understand the denary, binary, and hexadecimal number systems.
    • (b) Convert between:
      • (i) positive denary and positive binary.
      • (ii) positive denary and positive hexadecimal.
      • (iii) positive hexadecimal and positive binary.
  • Hexadecimal Usage: Understand how and why hexadecimal is used as a beneficial method of data representation.
  • Binary Addition:
    • (a) Add two positive 8-bit binary integers.
    • (b) Understand the concept of overflow and why it occurs in binary addition.
  • Binary Shifts: Perform a logical binary shift on a positive 8-bit binary integer and understand the effect this has on the positive binary integer.
  • Two's Complement: Use the two's complement number system to represent positive and negative 8-bit binary integers.

Notes & Guidance

  • Hexadecimal: Commonly used in error codes and MAC addresses. It is used because it is shorter and easier for humans to read than long strings of binary.
  • Overflow: Occurs if the result of an addition is greater than 255 (the maximum value an 8-bit register can hold).
  • Shifts:
    • Left shift = Multiply by 2.
    • Right shift = Divide by 2.
    • Bits shifted out are lost; zeros are shifted in to fill the gap.
  • Two's Complement: You must be able to convert positive/negative integers to/from 8-bit two's complement.

1.2 Text, Sound and Images

What you need to know:

  • Text: Understand how and why a computer represents text and the use of character sets, including ASCII and Unicode.
  • Sound: Understand how and why a computer represents sound, including the effects of the sample rate and sample resolution.
  • Images: Understand how and why a computer represents an image, including the effects of the resolution and colour depth.

Notes & Guidance

  • Text: Unicode allows for a much greater range of characters (different languages, emojis) than ASCII but requires more bits per character.
  • Sound:
    • Sample rate = number of samples taken per second.
    • Sample resolution = number of bits used per sample.
    • Trade-off: As these increase, accuracy improves, but file size increases.
  • Images:
    • Image = a series of pixels (picture elements).
    • Resolution = total number of pixels.
    • Colour depth = number of bits used to represent the colour of each pixel.
    • Trade-off: As resolution and colour depth increase, quality improves, but file size increases.

1.3 Data Storage and Compression

What you need to know:

  • Measurement: Understand how data storage is measured.
  • File Size Calculation: Calculate the file size of an image file and a sound file, using information given.
  • Compression Purpose: Understand the purpose of and need for data compression.
  • Compression Methods: Understand how files are compressed using lossy and lossless compression methods.

Notes & Guidance

  • Units: bit, nibble, byte, kibibyte (KiB), mebibyte (MiB), gibibyte (GiB), tebibyte (TiB), pebibyte (PiB), exbibyte (EiB).
  • Calculations (CRITICAL): You must use the measurement of 1024, not 1000. (e.g., 1 KiB = 1024 bytes).
  • Compression Need: Reduces bandwidth usage, saves storage space, and reduces transmission time.
  • Lossy: Permanently removes data to reduce size (e.g., reducing resolution, colour depth, or sample rate).
  • Lossless: Reduces size without permanent data loss (e.g., Run Length Encoding (RLE)).

Exam Expectations: What You Need to Do

For the Cambridge IGCSE (0478) exam, you must be able to:

  • Convert Fluidly: Switch numbers between binary, denary, and hexadecimal formats without a calculator.
  • Calculate Accurately:
    • Perform binary addition (handling overflows) and logical shifts.
    • Calculate file sizes for images and sound. Crucial: You must use 1024 (not 1000) when converting between units like KiB, MiB, and GiB.
  • Explain the "Why": Justify why hexadecimal is used (e.g., for MAC addresses or error codes) and why specific compression methods are chosen for different file types.
  • Analyse Trade-offs: Describe how changing the resolution, colour depth, or sample rate affects both the quality of the media and its file size.
