What Is Binary?

Why Hardware Loves It
  • Transistors are stable in two states (low/high voltage).
  • Noise-tolerant and easy to store as charges or magnetic domains.
  • Unsigned range doubles with each extra bit (n bits -> 0 to 2^n - 1).
Clarify

Binary is chosen for reliability, not because it is the easiest for humans to read.

Step 2: 156 >= 128? Yes -> bit = 1, remainder 28

Step 3: 28 >= 64? No -> bit = 0

Step 4: 28 >= 32? No -> bit = 0

Step 5: 28 >= 16? Yes -> bit = 1, remainder 12

Step 6: 12 >= 8? Yes -> bit = 1, remainder 4

Step 7: 4 >= 4? Yes -> bit = 1, remainder 0

Step 8: Remaining bits (2,1) = 0 0 -> 100111002

8 bits hold 0-255 (unsigned). More bits = more range.

Clarify: Binary Understanding

Why does adding more bits increase the maximum value?

Because each new bit adds the next power-of-two place value, doubling the unsigned range (n bits -> 0 to 2^n - 1).

Example: 8 bits max 255, 9 bits max 511.

What Is Hexadecimal?

Why It Exists
  • Human-friendly shorthand for long binary strings
  • Maps neatly to 4-bit nibbles (1 hex digit = 4 bits)
  • Reduces copying mistakes in addresses, colour codes, and error IDs
Where You'll See It

MAC addresses, memory/register dumps, error codes (0x8007...), and colour values (#3FA7D6).

Binary to hex: Group bits into nibbles from the right.

Hex to binary: Expand each hex digit to 4 bits.

How do we convert 9410 to hex?

Step 1: Divide by 16 -> 94 / 16 = 5 remainder 14

Map the remainder: 14 -> E in hex.

Step 2: Write quotient then remainder -> 5E16

If the quotient >= 16, divide again for an extra hex digit.

Check: (5 x 16) + 14 = 80 + 14 = 94

Write hex digits in order of quotients -> remainders from top to bottom.

Clarify: Hex Understanding

Why is hex preferred over binary for humans?

Because it is shorter to read/write and maps cleanly to 4-bit groups, reducing mistakes.

Binary Addition (8-bit)

Add column by column; 1+1 = 0 carry 1.

Check: 85 + 57 = 14210 -> 100011102

Overflow

Example: 11110000 + 11110000

Work: result bits = 11100000 with a carry out of the 8th bit

Meaning: Needs 9 bits -> overflow (too big for 8-bit register)

Range reminder: 8-bit unsigned max = 255; any sum above that overflows.

What Are Logical Shifts?

Why It Exists
  • Fast multiply/divide by 2 for unsigned values
  • Prep values before masking/combining bits
Clarify

Left shift multiplies by 2 per shift; right shift divides by 2 and truncates.

Shifted-out bits are lost; sign is not preserved (use arithmetic shift for signed values).

What Is Two's Complement?

Why It Exists
  • Enables addition/subtraction with the same adder hardware
  • Only one representation of zero (no +0/-0 duplicate)
Range

8-bit: -128 to +127. MSB = 1 means negative; magnitude is found by converting back.

How do we represent -25 in two's complement (8-bit)?

Press Down to see each step.

Step 1: Write +25 in 8 bits (pad with leading zeros) -> 00011001

Step 2: Invert bits -> 11100110

Step 3: Add 1 -> 11100111

Check: Convert back: invert (00011000) +1 -> 00011001 = 25 -> sign bit 1 -> -25

To read a negative: invert + add 1 again.

Try It: Two's Complement

Question: What is -58 in 8-bit two's complement?

Answer: 00111010 -> invert 11000101 -> +1 = 11000110

Clarify: Two's Complement

Why is there only one zero in two's complement?

Because the inversion+1 method makes the all-zeros pattern unique; no negative zero.

True or False?

Answer these misconception checks in your book:

  1. "After 9 in hex the next digit is 10."
  2. "Overflow happens when a sum needs more bits than the register size."
  3. "A logical right shift keeps the remainder when dividing by 2."
  4. "Two's complement negatives are made by inverting bits then adding 1."

Press Down to reveal answers

Question 1

"After 9 in hex the next digit is 10."

Answer: False

Hex continues with A, B, C, D, E, F before rolling over.

Question 2

"Overflow happens when a sum needs more bits than the register size."

Answer: True

Carry out of the most significant bit indicates overflow in fixed-width arithmetic.

Question 3

"A logical right shift keeps the remainder when dividing by 2."

Answer: False

Logical right shifts truncate; shifted-out bits vanish and zeros fill on the left.

Question 4

"Two's complement negatives are made by inverting bits then adding 1."

Answer: True

That exact invert + 1 algorithm creates the negative; subtracting 1 would be incorrect.
