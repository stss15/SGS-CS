# 7.5 Validation and Verification - Revision Artifact Prompts

## Concept: Validation Checks
- **Type**: Icon List / Infographic
- **Description**: The different methods to ensure data is reasonable.
- **Prompt**: Create an infographic for "Types of Validation".
  - **Range Check**: Icon of a Ruler. Text: "Is it within limits? (e.g., Age 0-120)."
  - **Length Check**: Icon of a Tape Measure. Text: "Is it the right length? (e.g., Password 8 chars)."
  - **Type Check**: Icon of `Abc` vs `123`. Text: "Is it the right data type? (e.g., Integer, not String)."
  - **Presence Check**: Icon of a Checkbox. Text: "Is it empty? (Required fields)."
  - **Format Check**: Icon of a License Plate. Text: "Does it match the pattern? (e.g., XX99 XX)."
  - **Check Digit**: Icon of a Barcode. Text: "Is the code valid? (Calculated extra digit)."

## Concept: Validation vs Verification
- **Type**: Comparison Visual
- **Description**: Clarifying the difference between the two V's.
- **Prompt**: Create a comparison of "Validation vs. Verification".
  - **Validation**:
    - Question: "Is the data REASONABLE?"
    - Method: Computer checks against rules (Range, Type, etc.).
    - Example: "Rejecting 'Age: 200'."
  - **Verification**:
    - Question: "Is the data CORRECT (matches source)?"
    - Method: Human check or Double Entry.
    - Example: "Typing password twice to confirm."
