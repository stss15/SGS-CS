# 7.2 Computer Systems and Design Methods - Revision Artifact Prompts

## Concept: Structure Diagrams
- **Type**: Hierarchical Tree Diagram
- **Description**: Visualizing Top-Down Design.
- **Prompt**: Create a structure diagram for an "Alarm App".
  - **Top Box**: "Alarm App System".
  - **Level 2**: Three branches:
    - "Set Alarm"
    - "Check Time"
    - "Sound Alarm"
  - **Level 3 (under Set Alarm)**: "Input Hour", "Input Minute", "Save Time".
  - **Annotation**: "Top-Down Design: Breaking a system into sub-systems."

## Concept: Flowchart Symbols
- **Type**: Legend / Key
- **Description**: The standard symbols used in flowcharts.
- **Prompt**: Create a "Flowchart Symbols Key".
  - **Start/Stop**: Oval/Rounded Rectangle. Label: "Terminator (Start/End)".
  - **Process**: Rectangle. Label: "Process (Calculation/Assignment)". Example: `Count = Count + 1`.
  - **Input/Output**: Parallelogram. Label: "Input / Output". Example: `INPUT Name`.
  - **Decision**: Diamond. Label: "Decision (Yes/No)". Example: `Is X > 10?`.
  - **Flow Line**: Arrow. Label: "Direction of Flow".

## Concept: Pseudocode Basics
- **Type**: Code Cheat Sheet
- **Description**: Common keywords and structures.
- **Prompt**: Create a "Pseudocode Cheat Sheet" visual.
  - **Input/Output**: `INPUT x`, `OUTPUT "Hello"`.
  - **Assignment**: `Total ← Total + 1` (Use arrow).
  - **Selection (IF)**:
    ```
    IF x > 10 THEN
      OUTPUT "High"
    ELSE
      OUTPUT "Low"
    ENDIF
    ```
  - **Selection (CASE)**:
    ```
    CASE OF Grade
      "A": OUTPUT "Top"
      "B": OUTPUT "Good"
      OTHERWISE OUTPUT "Fail"
    ENDCASE
    ```
  - **Iteration (Loops)**:
    - `FOR i ← 1 TO 10 ... NEXT i` (Fixed count)
    - `REPEAT ... UNTIL x > 10` (Run at least once)
    - `WHILE x < 10 DO ... ENDWHILE` (Check first)
