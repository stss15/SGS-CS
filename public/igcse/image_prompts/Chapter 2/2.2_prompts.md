# 2.2 Methods of Error Detection - Revision Artifact Prompts

## Concept: Parity Check
- **Type**: Logic Visual
- **Description**: Explaining the concept of Even and Odd parity with a clear example.
- **Prompt**: Create a visual explanation of "Parity Check".
  - **Visual**: An 8-bit byte `1 0 1 1 0 0 1 [?]`. The last bit is the Parity Bit.
  - **Even Parity Example**: "Count the 1s: 1, 2, 3, 4 (Even). Parity Bit = 0".
  - **Odd Parity Example**: "Count the 1s: 1, 2, 3, 4 (Even). Parity Bit = 1 (to make total odd)".
  - **Note**: "Parity checks detect simple errors (flipped bits)."

## Concept: Parity Block (2D Check)
- **Type**: Grid Diagram
- **Description**: Showing how a 2D parity check can locate the exact position of an error.
- **Prompt**: Create a 2D grid diagram (4x4).
  - **Grid**: Fill with 1s and 0s.
  - **Rows**: Show a "Row Parity" bit at the end of each row.
  - **Columns**: Show a "Column Parity" bit at the bottom of each column.
  - **Error**: Highlight one bit in the middle in RED.
  - **Detection**: Highlight the Row Parity and Column Parity for that bit in RED.
  - **Conclusion**: "Intersection of invalid Row and Column identifies the error bit."

## Concept: Check Digit (ISBN)
- **Type**: Formula Flow
- **Description**: A step-by-step guide to the ISBN-13 check digit algorithm.
- **Prompt**: Create a flow diagram for "ISBN-13 Check Digit".
  - **Input**: `9 7 8 0 3 4 0 9 8 3 8 2`.
  - **Step 1**: "Add Odd positions".
  - **Step 2**: "Add Even positions and Multiply by 3".
  - **Step 3**: "Add results + Divide by 10".
  - **Step 4**: "10 - Remainder = Check Digit".
  - **Purpose**: "Detects transposition errors and incorrect digits."

## Concept: Automatic Repeat Request (ARQ)
- **Type**: Timeline Flowchart
- **Description**: The sequence of events in an ARQ system.
- **Prompt**: Create a timeline flowchart.
  - **Sender**: "Sends Data Packet". -> "Starts Timer".
  - **Scenario A (Success)**: Receiver sends "ACK (Acknowledgement)". Timer stops. Next packet sent.
  - **Scenario B (Failure)**: Timer reaches "Timeout". No ACK received. Sender "Resends Data".
  - **Scenario C (Error)**: Receiver detects error. Sends "NACK (Negative Acknowledgement)". Sender "Resends Data".

## Concept: Echo Check
- **Type**: Cycle Diagram
- **Description**: The simple loop of sending data, returning it, and comparing.
- **Prompt**: Create a circular process diagram. Step 1: "Computer A sends Data (1011)". Arrow to "Server". Step 2: "Server returns Data (1011)". Arrow back to "Computer A". Step 3: "Computer A compares Original vs Returned". Show a green checkmark for "Match". Simple, flat vector graphics.
