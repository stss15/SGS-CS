# 1.3 Data Storage and Compression - Revision Artifact Prompts

## Concept: Data Storage Units
- **Type**: Hierarchy Diagram
- **Description**: A vertical ladder showing the relationship between data units.
- **Prompt**: Create a vertical ladder diagram for "Data Units".
  - **Steps (Bottom to Top)**: "Bit (b)", "Nibble (4 bits)", "Byte (B)", "Kibibyte (KiB)", "Mebibyte (MiB)", "Gibibyte (GiB)", "Tebibyte (TiB)".
  - **Annotations**:
    - Arrow between Bit and Byte: "x 8".
    - Arrows between all other steps pointing DOWN labeled "x 1024".
    - Arrows between all other steps pointing UP labeled "/ 1024".
  - **Style**: Clean, technical, easy to read.

## Concept: Calculating File Sizes
- **Type**: Formula Cheat Sheet
- **Description**: A visual guide to calculating file sizes for images and sound.
- **Prompt**: Create a revision sheet titled "File Size Formulas".
  - **Section 1: Images**:
    - Formula: `File Size = Width x Height x Colour Depth`.
    - Note: "Divide by 8 to get Bytes".
    - Worked Example: `100px x 100px x 24-bit = 240,000 bits / 8 = 30,000 Bytes`.
  - **Section 2: Sound**:
    - Formula: `File Size = Sample Rate x Resolution x Channels x Duration`.
    - Note: "Divide by 8 to get Bytes".
    - Worked Example: `44,100Hz x 16-bit x 2 (Stereo) x 10s = 14,112,000 bits`.

## Concept: Compression Types (Lossy vs Lossless)
- **Type**: Comparison Chart
- **Description**: Contrasting the two main types of compression.
- **Prompt**: Create a split-screen comparison chart.
  - **Left (Lossy)**:
    - Title: "Lossy Compression".
    - Definition: "Permanently removes data to reduce size."
    - Visual: A photo becoming slightly pixelated.
    - Examples: "JPEG (Images), MP3 (Audio), MP4 (Video)".
  - **Right (Lossless)**:
    - Title: "Lossless Compression".
    - Definition: "Original data is fully restored."
    - Visual: A zipper closing/opening.
    - Examples: "RLE, ZIP, PNG".

## Concept: Run Length Encoding (RLE)
- **Type**: Process Visual
- **Description**: Demonstrating how RLE compresses repeating data.
- **Prompt**: Create a visual explanation of "Run Length Encoding".
  - **Input**: A row of colored blocks: `Red, Red, Red, Red, Green, Green, Blue`.
  - **Process Arrow**: "Compress".
  - **Output**: The data pairs: `4 Red, 2 Green, 1 Blue`.
  - **Explanation Box**: "RLE stores 'Count-Value' pairs. It is most effective for data with long runs of repeating values (e.g., simple icons)."
