# 9.1 Databases and SQL - Revision Artifact Prompts

## Concept: Database Structure
- **Type**: Annotated Diagram
- **Description**: Visualizing the components of a database table.
- **Prompt**: Create an annotated diagram of a "Database Table".
  - **The Table**: A grid with data about "Superheroes".
  - **Columns (Fields)**: Vertical highlight. Label: "Field / Attribute". Example: "Name", "Power", "Age".
  - **Rows (Records)**: Horizontal highlight. Label: "Record". Example: "Clark Kent, Flight, 35".
  - **Cell**: Single box. Label: "Data Item".

## Concept: Primary Key
- **Type**: Visual Analogy
- **Description**: The unique identifier for a record.
- **Prompt**: Create a visual for "Primary Key".
  - **Visual**: A row of houses.
  - **Problem**: Two houses look identical (same color, same shape).
  - **Solution**: A unique "House Number" plate on each door.
  - **Icon**: A Gold Key next to the "ID" field in a table.
  - **Text**: "Unique Identifier. No duplicates allowed."

## Concept: SQL Commands
- **Type**: Code Visual / Cheat Sheet
- **Description**: The basic structure of an SQL query.
- **Prompt**: Create a visual breakdown of an "SQL Query".
  - **SELECT**: Icon of a Hand picking items. Text: "Which FIELDS to show?"
  - **FROM**: Icon of a Table/File. Text: "Which TABLE to look in?"
  - **WHERE**: Icon of a Filter/Funnel. Text: "Which RECORDS match the condition?"
  - **ORDER BY**: Icon of A-Z steps. Text: "How to SORT the results?"
  - **Example**:
    ```sql
    SELECT Name, Age
    FROM Students
    WHERE Age > 15
    ORDER BY Name
    ```

## Concept: SQL Operators
- **Type**: Symbol Chart
- **Description**: Common operators used in WHERE clauses.
- **Prompt**: Create a chart for "SQL Operators".
  - **Standard**: `=`, `>`, `<`, `>=`, `<=`, `<>` (Not Equal).
  - **BETWEEN**: `WHERE Age BETWEEN 10 AND 20`.
  - **LIKE**: `WHERE Name LIKE 'S%'` (Starts with S).
  - **IN**: `WHERE Color IN ('Red', 'Blue')`.
  - **AND/OR**: `WHERE Age > 10 AND Height > 150`.
