# 4.2 Programming Languages and IDEs - Revision Artifact Prompts

## Concept: Language Levels
- **Type**: Staircase Diagram
- **Description**: From Human to Machine.
- **Prompt**: Create a "Programming Language Levels" staircase.
  - **Top Step**: "High-Level Language (Python, Java)". Text: "Close to English. Portable. Easy to debug. Problem-oriented."
  - **Middle Step**: "Assembly Language". Text: "Mnemonics (LDA, ADD). Hardware specific. 1-to-1 with Machine Code."
  - **Bottom Step**: "Machine Code (Binary)". Text: "1s and 0s. Directly executed by CPU. Hard to understand."
  - **Arrow**: Downwards arrow labeled "Translation Required".

## Concept: Translators (Compiler vs Interpreter)
- **Type**: Comparison Battle Card
- **Description**: Pros and Cons of each.
- **Prompt**: Create a comparison card for "Compiler vs Interpreter".
  - **Compiler**:
    - "Translates WHOLE code at once."
    - "Creates Executable (.exe)."
    - "Fast execution."
    - "Source code not needed to run."
    - "Harder to debug (List of errors at end)."
  - **Interpreter**:
    - "Translates LINE BY LINE."
    - "No Executable created."
    - "Slower execution."
    - "Source code needed to run."
    - "Easier to debug (Stops at error)."

## Concept: The IDE (Integrated Development Environment)
- **Type**: Interface Mockup
- **Description**: Labeling the parts of an IDE.
- **Prompt**: Create a labeled screenshot of an "IDE Interface".
  - **Main Window**: "Code Editor". Label: "Syntax Highlighting (Prettyprinting), Auto-completion, Auto-correction."
  - **Bottom Window**: "Output / Console". Label: "Shows results of program."
  - **Side Panel**: "File Explorer / Project Manager".
  - **Tools**: "Translator (Run Button)", "Debugger (Breakpoints, Variable Watch)."
  - **Definition**: "Software suite for writing and testing programs."

## Concept: Translation Process
- **Type**: Process Flow
- **Description**: How source code becomes a running program.
- **Prompt**: Create a flow diagram for "Translation".
  - **Path A (Compiler)**: Source Code -> Compiler -> Object Code (Machine Code) -> Processor.
  - **Path B (Interpreter)**: Source Code -> Interpreter -> Processor (Line by Line).
  - **Path C (Assembler)**: Assembly Code -> Assembler -> Machine Code -> Processor.
