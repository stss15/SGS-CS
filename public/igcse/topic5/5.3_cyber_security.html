Question 1

What is the difference between Digital Currency and Cryptocurrency?

Answer:

Digital Currency is centralized (banks control it). Cryptocurrency is decentralized (community/algorithm controls it).

Question 2

What makes blockchain data immutable?

Answer:

Each block contains the hash of the previous block. Changing data breaks the chain.

Question 3

What does "decentralized" mean?

Answer:

No single authority controls it. It's distributed across many computers.

Deep Dive: The Attacks

Hacking & Brute Force

Brute Force

The Method: Guessing every password combination.

"123456" "password" "qwerty"

Defense: Strong passwords, lockout after 3 tries.

DDoS

Distributed Denial of Service

Flooding a server with traffic to crash it.

Analogy: Like thousands of people calling a phone line at once.

Malware Types

Virus vs Worm

Virus: Needs a host file. User must run it.

Like a parasite.

Worm: Standalone. Spreads itself automatically.

Like a self-replicating robot.

Ransomware

Encrypts your files.

Demands money (Bitcoin) to unlock them.

Example: "Your files are locked. Pay $500 to unlock."

Trojan Horse: Disguised as legitimate software (e.g., "Free Game").

Opens a backdoor for hackers once installed.

Phishing vs Pharming

Social Engineering

Phishing

Fake emails (e.g., "Your bank account is locked").

Goal: Steal credentials.

Signs: Bad spelling, urgent tone, generic greeting.

Like a fake letter in your mailbox.

Pharming

Malicious code redirects you to a fake website even if you type the right URL.

Mechanism: DNS Poisoning.

Like road signs pointing to the wrong place.

The Human Factor: Exploiting fear, curiosity, or trust.

"You won a prize!" or "Urgent Security Alert!"

Deep Dive: Defense Strategies

Authentication Methods

Two-Step Verification

Authentication: Proving Who You Are

Passwords

Something you know.

Strong: Mixed case, numbers, symbols.

Like a secret code.

Biometrics

Something you are.

Fingerprint, Face ID, Retina.

Pros: Unique, can't lose it.

Cons: Privacy concerns, can't reset it.

2FA

Two-Factor Authentication.

Password + Phone Code.

Like needing two keys to open a safe.

Data Access Levels

Technical Defenses

Firewall

Monitors traffic in/out.

Blocks unauthorized access.

Like a security guard checking IDs.

Encryption

Scrambling data so it's unreadable.

Important: Does NOT stop interception, but makes the data useless to the thief.

Like writing in a secret code.

Anti-Malware

Scans for known virus signatures and suspicious behavior.

True or False?

Misconception Check

  1. "A strong password prevents hacking."
  2. "Pharming requires the user to click a link."
  3. "Encryption stops data from being stolen."
  4. "Biometrics are 100% secure."

Press Down to reveal answers

Question 1

"A strong password prevents hacking."

Answer: False

It helps against Brute Force, but not Phishing or Keyloggers.

Question 2

"Pharming requires the user to click a link."

Answer: False

That's Phishing. Pharming redirects you automatically (DNS poisoning).

Question 3

"Encryption stops data from being stolen."

Answer: False

It stops it from being read. It can still be stolen or deleted.

Question 4

"Biometrics are 100% secure."

Answer: False

Fingerprints can be lifted, faces can be spoofed (though it's hard).
