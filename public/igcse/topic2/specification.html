Topic 2: Data Transmission

Complete Specification & Learning Guide

Cambridge IGCSE Computer Science 0478 (2026–2028)

Moving Data Around the World

Data is frequently transferred from one device to another. The two devices could be in the same building or thousands of kilometres away. Irrespective of the distance travelled, the transmission of data needs to be considered carefully. In this unit, we explore how data travels, how we catch errors during the journey, and how we keep it safe from prying eyes.

What You Will Learn

In this section, we cover the essential mechanisms of communication:

  • Data Transmission: How data is broken into packets, routed across networks, and the different physical methods of sending bits (Serial vs Parallel, Simplex vs Duplex).
  • Error Detection: The clever methods computers use to ensure the data received is exactly what was sent (Parity, Checksums, ARQ).
  • Encryption: How we use mathematics (Symmetric and Asymmetric keys) to make data unreadable to unauthorised users.

2.1 Types and Methods of Data Transmission

What you need to know:

  • Data Packets:
    • (a) Understand that data is broken down into packets to be transmitted.
    • (b) Describe the structure of a packet (Packet Header, Payload, Trailer).
    • (c) Describe the process of packet switching.
  • Transmission Methods:
    • (a) Describe how data is transmitted from one device to another using different methods:
      • Serial vs Parallel
      • Simplex vs Half-duplex vs Full-duplex
    • (b) Explain the suitability of each method of data transmission for a given scenario.
  • USB Interface: Understand the Universal Serial Bus (USB) interface and explain how it is used to transmit data.

Notes & Guidance

  • Packet Structure:
    • Header: Contains destination address, packet number, and originator's address.
    • Payload: The actual data being sent.
    • Trailer: End of packet marker and error checking data.
  • Packet Switching:
    • Data is broken into packets.
    • Each packet may take a different route through the network.
    • A router controls the route a packet takes.
    • Packets may arrive out of order and must be reordered upon arrival.
  • Transmission Terminology:
    • Serial: One bit at a time over a single wire (reliable over long distances).
    • Parallel: Multiple bits simultaneously over multiple wires (faster but prone to interference/skew over long distances).
    • Simplex: One direction only (e.g., radio broadcast).
    • Half-duplex: Both directions, but not at the same time (e.g., walkie-talkie).
    • Full-duplex: Both directions simultaneously (e.g., telephone call).

2.2 Methods of Error Detection

What you need to know:

  • The Need for Checks: Understand the need to check for errors after data transmission and how these errors can occur (interference, data loss, data gain, data change).
  • Transmission Error Detection: Describe the processes involved in:
    • Parity checks (odd and even)
    • Checksums
    • Echo checks
  • Data Entry Error Detection: Describe how a check digit is used to detect errors in data entry (e.g., ISBNs and barcodes).
  • ARQ: Describe how an Automatic Repeat Query (ARQ) can be used to establish that data is received without error.

Notes & Guidance

  • Parity Check: Can be a parity bit (byte level) or a parity block check (checking a block of data). Note that parity checks cannot detect if an even number of bits have changed (transposition error).
  • ARQ: Uses positive/negative acknowledgements (ACK/NACK) and timeouts. If no acknowledgement is received before the timeout, the data is resent automatically.
  • Check Digits: Used to spot human error when typing in numbers (like swapping two digits or missing a digit).

2.3 Encryption

What you need to know:

  • Purpose: Understand the need for and purpose of encryption when transmitting data (confidentiality).
  • Methods: Understand how data is encrypted using:
    • Symmetric encryption
    • Asymmetric encryption

Notes & Guidance

  • Symmetric Encryption: Uses a single secret key for both encryption and decryption. Fast, but the key must be shared securely.
  • Asymmetric Encryption: Uses a pair of keys (Public Key and Private Key).
    • Public Key: Used to encrypt data (available to everyone).
    • Private Key: Used to decrypt data (kept secret by the receiver).

Exam Expectations: What You Need to Do

For the Cambridge IGCSE (0478) exam, you must be able to:

  • Describe Processes: Clearly describe how packet switching works, including the role of the router, and how ARQ handles timeouts.
  • Justify Choices: Explain why a specific transmission method (e.g., Serial vs Parallel) is suitable for a given scenario (e.g., sending data to a printer vs internal CPU data transfer).
  • Calculate Parity: Given a byte of data, determine the correct parity bit for odd or even parity.
  • Explain Encryption: Distinguish clearly between symmetric and asymmetric encryption, specifically mentioning the role of public and private keys.
