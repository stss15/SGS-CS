# Slide Plan: 2.2 Error Checking

**Goal:** Explain how computers detect errors in data transmission (Parity, Checksum, Echo Check, ARQ).

**Pedagogy:** Question First. Focus on "Trust but Verify".

---

## 1. Title Slide
- **Type:** Title
- **Content:**
  - Main Title: 2.2 Error Checking
  - Sub-title: Topic 2: Data Transmission
- **Notes:** Standard title layout.

## 2. Hook / Big Question
- **Type:** Concept (Horizontal)
- **Content:**
  - **Question:** What happens if a 0 flips to a 1?
  - **Visual:** A corrupted image or a crashed game.
  - **Key Idea:** Interference (noise) can corrupt data. We need to check it.

## 3. Parity Check
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** The Parity Bit
  - **Concept:** Adding an extra bit to make the total number of 1s either Even or Odd.
  - **Even Parity:** Count 1s. If 3, add a 1 (Total 4). If 4, add 0 (Total 4).

### 3.1 Vertical Deep Dive: The Parity Block
- **Type:** Vertical Stack
- **Slide 1 (Simple Parity):**
  - Problem: If 2 bits flip, the check passes! (Error not detected).
- **Slide 2 (Parity Block):**
  - Arranging data in a grid.
  - Checking rows AND columns.
  - **Benefit:** Can identify exactly *which* bit flipped and **correct** it.

## 4. Checksum
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** The Checksum
  - **Method:**
    1. Calculate a value from the data (Maths).
    2. Send data + value.
    3. Receiver recalculates.
    4. If values match -> OK. If not -> Error.
  - **Use:** Credit Cards, ISBNs, File downloads.

## 5. Echo Check
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** Echo Check
  - **Method:** Receiver sends the data *back* to the sender. Sender compares.
  - **Problem:** If the echo is wrong, was the error in the sending or the return? Inefficient.

## 6. ARQ (Automatic Repeat Request)
- **Type:** Concept (Horizontal)
- **Content:**
  - **Title:** ARQ
  - **Components:**
    1. **Acknowledgement (ACK):** Receiver confirms receipt.
    2. **Timeout:** Sender waits. If no ACK, resend.
  - **Analogy:** "Did you get that?" ... "Did you get that?"

## 7. Plenary: True or False?
- **Type:** Quiz (Vertical Stack)
- **Slide 1 (Intro):** Misconception Check.
- **Slide 2 (Q1):** "A parity check can detect if 2 bits flip."
  - **Answer:** False. It will still look even/odd.
- **Slide 3 (Q2):** "A parity block check can correct errors."
  - **Answer:** True. By cross-referencing row and column.
- **Slide 4 (Q3):** "ARQ relies on the receiver sending the data back."
  - **Answer:** False. That's Echo Check. ARQ sends an Acknowledgement (ACK).
